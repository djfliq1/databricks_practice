Section 16 — Delta Lake Optimization
FeatureInputProcess / LogicOutput
OPTIMIZE (bin-packing)Delta table with many small filesMerges small files into ~1GB target file size — idempotentFewer, larger Parquet files — faster scan performance
OPTIMIZE ZORDER BYDelta table + column(s) to co-locateReorders data within files by column value — NOT idempotentData physically sorted to minimize files scanned per predicate
Liquid ClusteringTable with CLUSTER BY definitionAdaptive, incremental clustering — replaces partitioningClustered files without full rewrite — idempotent, flexible key changes
VACUUMDelta table + retention period (default 7 days)Deletes unreferenced Parquet files older than retention windowReclaimed cloud storage — older time travel versions no longer accessible
DEEP CLONESource Delta tableFull data + transaction log copy — re-run is incrementalIndependent Delta table clone — source VACUUM-safe
SHALLOW CLONESource Delta tableCopies only transaction log metadata — data files referenced via pointersLightweight clone — BREAKS if source is VACUUMed
Time TravelVERSION AS OF or TIMESTAMP AS OFReads specific Delta log version — no data movementRead-only snapshot of table at that point in time
RESTORE TABLETarget version or timestampRolls table back — adds new transaction log entry pointing to prior versionTable in prior state — rollback is a new version, not destructive
