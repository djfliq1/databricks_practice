The Cortana Overview

When dealing with SQL tasks and large arrays, the solution is table-to-table data flow rather than JSON passing. SQL tasks can't return complex JSON, so large arrays flow through intermediate tables, views, and direct table references. This is the most efficient pattern for SQL-heavy pipelines.

🎯 Pattern 1: SQL Task → Large Result Table → Downstream SQL

SQL Task: Create Large Analysis Table

-- notebooks/01_large_analysis_sql.sql
-- SQL task that creates large result table instead of returning JSON

-- Create comprehensive product analysis table with large feature arrays
CREATE OR REPLACE TABLE databricks_clar.temp.large_product_analysis AS
WITH base_products AS (
  SELECT 
    product_id,
    product_name,
    category,
    price,
    processDate,
    -- Generate large feature arrays using SQL functions
    ARRAY(
      ROUND(price * RAND(), 2),
      ROUND(price * RAND() * 0.8, 2),
      ROUND(price * RAND() * 1.2, 2),
      ROUND(price * RAND() * 0.9, 2),
      ROUND(price * RAND() * 1.1, 2),
      ROUND(price * RAND() * 0.7, 2),
      ROUND(price * RAND() * 1.3, 2),
      ROUND(price * RAND() * 0.6, 2),
      ROUND(price * RAND() * 1.4, 2),
      ROUND(price * RAND() * 0.5, 2)
    ) as price_predictions,

    -- Large metadata map
    MAP(
      'supplier_rating', ROUND(RAND() * 5, 2),
      'quality_score', ROUND(RAND() * 10, 2),
      'inventory_turnover', ROUND(RAND() * 50, 2),
      'seasonal_factor', ROUND(RAND() * 3, 2),
      'market_share', ROUND(RAND() * 100, 2),
      'competitor_price_1', ROUND(price * (0.8 + RAND() * 0.4), 2),
      'competitor_price_2', ROUND(price * (0.9 + RAND() * 0.2), 2),
      'competitor_price_3', ROUND(price * (1.0 + RAND() * 0.3), 2),
      'demand_forecast_q1', ROUND(RAND() * 1000, 0),
      'demand_forecast_q2', ROUND(RAND() * 1200, 0),
      'demand_forecast_q3', ROUND(RAND() * 800, 0),
      'demand_forecast_q4', ROUND(RAND() * 1500, 0)
    ) as market_intelligence,

    -- Complex feature engineering
    STRUCT(
      CASE WHEN price > 250 THEN 'premium' ELSE 'standard' END as tier,
      ROUND(price / (1 + RAND()), 2) as cost_estimate,
      DATE_ADD(CURRENT_DATE(), CAST(RAND() * 365 AS INT)) as next_review_date,
      ARRAY(
        STRUCT('feature_a' as name, ROUND(RAND() * 100, 2) as score, 'high' as importance),
        STRUCT('feature_b' as name, ROUND(RAND() * 100, 2) as score, 'medium' as importance),
        STRUCT('feature_c' as name, ROUND(RAND() * 100, 2) as score, 'low' as importance),
        STRUCT('feature_d' as name, ROUND(RAND() * 100, 2) as score, 'high' as importance),
        STRUCT('feature_e' as name, ROUND(RAND() * 100, 2) as score, 'medium' as importance)
      ) as detailed_features
    ) as enrichment_data,

    -- Processing metadata
    CURRENT_TIMESTAMP() as analysis_timestamp,
    UUID() as batch_id

  FROM databricks_clar.bronze.products_bronze
),

-- Cross join to simulate larger dataset (your 100 products → 500+ records)
expanded_analysis AS (
  SELECT 
    CONCAT(product_id, '_', scenario.id) as analysis_id,
    product_id,
    product_name,
    category,
    price,
    scenario.scenario_name,
    scenario.price_multiplier,

    -- Scenario-adjusted predictions
    TRANSFORM(price_predictions, x -> ROUND(x * scenario.price_multiplier, 2)) as adjusted_predictions,
    market_intelligence,
    enrichment_data,
    analysis_timestamp,
    batch_id

  FROM base_products
  CROSS JOIN (
    SELECT 1 as id, 'baseline' as scenario_name, 1.0 as price_multiplier
    UNION ALL SELECT 2 as id, 'optimistic' as scenario_name, 1.2 as price_multiplier  
    UNION ALL SELECT 3 as id, 'pessimistic' as scenario_name, 0.8 as price_multiplier
    UNION ALL SELECT 4 as id, 'peak_season' as scenario_name, 1.5 as price_multiplier
    UNION ALL SELECT 5 as id, 'off_season' as scenario_name, 0.6 as price_multiplier
  ) as scenario
)

SELECT 
  analysis_id,
  product_id,
  product_name,
  category,
  price,
  scenario_name,
  price_multiplier,
  adjusted_predictions,
  market_intelligence,
  enrichment_data,

  -- Additional computed columns for large array processing
  SIZE(adjusted_predictions) as prediction_count,

  -- Extract key metrics from complex structures
  enrichment_data.tier as product_tier,
  enrichment_data.cost_estimate,
  enrichment_data.next_review_date,

  -- Array aggregations
  AGGREGATE(adjusted_predictions, 0.0, (acc, x) -> acc + x) / SIZE(adjusted_predictions) as avg_prediction,
  ARRAY_MAX(adjusted_predictions) as max_prediction,
  ARRAY_MIN(adjusted_predictions) as min_prediction,

  -- Map value extractions
  market_intelligence['supplier_rating'] as supplier_rating,
  market_intelligence['quality_score'] as quality_score,
  market_intelligence['inventory_turnover'] as inventory_turnover,

  analysis_timestamp,
  batch_id

FROM expanded_analysis;

-- Create summary metadata table for downstream tasks
CREATE OR REPLACE TABLE databricks_clar.temp.analysis_metadata AS
SELECT 
  COUNT(*) as total_analysis_records,
  COUNT(DISTINCT product_id) as unique_products,
  COUNT(DISTINCT scenario_name) as scenarios_analyzed,
  COUNT(DISTINCT category) as categories_covered,
  ROUND(AVG(price), 2) as avg_base_price,
  ROUND(AVG(avg_prediction), 2) as avg_prediction_price,
  MIN(analysis_timestamp) as analysis_start,
  MAX(analysis_timestamp) as analysis_end,
  FIRST(batch_id) as batch_id
FROM databricks_clar.temp.large_product_analysis;

-- Output summary for monitoring (this would be captured in SQL warehouse logs)
SELECT 
  CONCAT('✅ Large analysis complete: ', total_analysis_records, ' records across ', 
         unique_products, ' products and ', scenarios_analyzed, ' scenarios') as status,
  batch_id
FROM databricks_clar.temp.analysis_metadata;

Notebook Task: Process Large SQL Table

# notebooks/02_process_large_sql_table.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Process Large SQL Table Results
# MAGIC **Reading and processing large arrays from SQL-generated tables**

# COMMAND ----------
# 🎯 SOLUTION: Read large data directly from SQL-generated table
df_large_analysis = spark.table("databricks_clar.temp.large_product_analysis")
df_metadata = spark.table("databricks_clar.temp.analysis_metadata")

# Get metadata for verification
metadata_row = df_metadata.collect()[0]
total_records = int(metadata_row["total_analysis_records"])
batch_id = metadata_row["batch_id"]
unique_products = int(metadata_row["unique_products"])

print(f"📊 Processing large analysis: {total_records} records from SQL task")
print(f"🔍 Batch ID: {batch_id}")
print(f"📦 Unique products: {unique_products}")

# COMMAND ----------
# Verify large array structure
sample_record = df_large_analysis.limit(1).collect()[0]

print(f"🔍 Sample Record Structure:")
print(f"   Analysis ID: {sample_record['analysis_id']}")
print(f"   Product: {sample_record['product_name']} ({sample_record['category']})")
print(f"   Scenario: {sample_record['scenario_name']}")
print(f"   Predictions Array Length: {sample_record['prediction_count']}")
print(f"   Market Intelligence Keys: {list(sample_record['market_intelligence'].keys())}")
print(f"   Enrichment Tier: {sample_record['product_tier']}")

# COMMAND ----------
# 🔥 EXAM PATTERN: Process large arrays without JSON serialization limits

# Category-specific processing of large arrays
categories = df_large_analysis.select("category").distinct().collect()

category_results = {}
for category_row in categories:
    category = category_row["category"]

    # Filter large dataset for this category
    df_category = df_large_analysis.filter(col("category") == category)
    category_count = df_category.count()

    print(f"Processing {category}: {category_count} analysis records...")

    # Complex array processing that would break JSON limits
    # 1. Prediction array analysis
    prediction_stats = df_category.select(
        explode(col("adjusted_predictions")).alias("prediction")
    ).agg(
        avg("prediction").alias("avg_prediction"),
        min("prediction").alias("min_prediction"),
        max("prediction").alias("max_prediction"),
        stddev("prediction").alias("stddev_prediction")
    ).collect()[0]

    # 2. Market intelligence aggregation
    market_stats = df_category.select(
        col("market_intelligence.supplier_rating").alias("supplier_rating"),
        col("market_intelligence.quality_score").alias("quality_score"),
        col("market_intelligence.inventory_turnover").alias("inventory_turnover")
    ).agg(
        avg("supplier_rating").alias("avg_supplier_rating"),
        avg("quality_score").alias("avg_quality_score"),
        avg("inventory_turnover").alias("avg_inventory_turnover")
    ).collect()[0]

    # 3. Feature analysis from complex structs
    feature_analysis = df_category.select(
        explode(col("enrichment_data.detailed_features")).alias("feature")
    ).select(
        col("feature.name").alias("feature_name"),
        col("feature.score").alias("feature_score"),
        col("feature.importance").alias("importance")
    ).groupBy("feature_name", "importance").agg(
        avg("feature_score").alias("avg_score"),
        count("*").alias("record_count")
    ).collect()

    # Create enhanced Silver table with processed arrays
    df_category_enhanced = df_category.select(
        "*",
        # Add prediction statistics as columns
        lit(float(prediction_stats["avg_prediction"])).alias("category_avg_prediction"),
        lit(float(prediction_stats["stddev_prediction"])).alias("category_prediction_volatility"),

        # Add market intelligence averages
        lit(float(market_stats["avg_supplier_rating"])).alias("category_avg_supplier_rating"),
        lit(float(market_stats["avg_quality_score"])).alias("category_avg_quality_score"),

        # Feature importance flags
        when(
            exists(
                col("enrichment_data.detailed_features"),
                lambda f: (f.importance == "high") & (f.score > 80)
            ), True
        ).otherwise(False).alias("has_high_importance_features"),

        # Processing metadata
        current_timestamp().alias("silver_processing_timestamp")
    )

    # Write category-specific enhanced table
    silver_table = f"databricks_clar.silver.large_analysis_{category.lower()}"
    df_category_enhanced.write.format("delta").mode("overwrite").saveAsTable(silver_table)

    # Store results for aggregation
    category_results[category] = {
        "analysis_records": category_count,
        "silver_table": silver_table,
        "prediction_stats": {
            "avg": float(prediction_stats["avg_prediction"]),
            "min": float(prediction_stats["min_prediction"]),
            "max": float(prediction_stats["max_prediction"]),
            "volatility": float(prediction_stats["stddev_prediction"])
        },
        "market_stats": {
            "supplier_rating": float(market_stats["avg_supplier_rating"]),
            "quality_score": float(market_stats["avg_quality_score"]),
            "inventory_turnover": float(market_stats["avg_inventory_turnover"])
        },
        "feature_analysis": [
            {
                "feature_name": row["feature_name"],
                "importance": row["importance"],
                "avg_score": float(row["avg_score"]),
                "record_count": int(row["record_count"])
            }
            for row in feature_analysis
        ]
    }

    print(f"✅ {category}: {category_count} records → {silver_table}")

# COMMAND ----------
# Clean up large temporary tables
spark.sql("DROP TABLE IF EXISTS databricks_clar.temp.large_product_analysis")
spark.sql("DROP TABLE IF EXISTS databricks_clar.temp.analysis_metadata")

print(f"🗑️  Cleaned up temporary SQL tables")

# COMMAND ----------
# Return processing summary (manageable JSON size)
import json

processing_output = {
    "processing_complete": True,
    "batch_id": batch_id,
    "total_analysis_records_processed": total_records,
    "unique_products_analyzed": unique_products,
    "categories_processed": len(category_results),
    "silver_tables_created": [r["silver_table"] for r in category_results.values()],
    "category_summary": {
        category: {
            "records": results["analysis_records"],
            "table": results["silver_table"],
            "avg_prediction": results["prediction_stats"]["avg"],
            "prediction_volatility": results["prediction_stats"]["volatility"]
        }
        for category, results in category_results.items()
    },
    "temp_tables_cleaned": True,

}

Downstream SQL Task: Process Large Arrays with SQL

-- notebooks/03_sql_large_array_processing.sql  
-- Pure SQL processing of large arrays from upstream SQL task

-- Process the large analysis table with SQL array functions
CREATE OR REPLACE TABLE databricks_clar.gold.prediction_analytics AS
WITH prediction_expanded AS (
  SELECT 
    analysis_id,
    product_id,
    product_name,
    category,
    scenario_name,
    price,

    -- Explode prediction arrays for SQL processing
    EXPLODE_OUTER(adjusted_predictions) as individual_prediction,

    -- Extract array statistics using SQL functions
    avg_prediction,
    max_prediction,
    min_prediction,

    -- Market intelligence processing
    supplier_rating,
    quality_score,
    inventory_turnover,

    -- Enrichment data processing
    product_tier,
    cost_estimate,

    analysis_timestamp,
    batch_id

  FROM databricks_clar.temp.large_product_analysis
),

prediction_rankings AS (
  SELECT 
    *,
    -- Rank predictions within each product across scenarios
    ROW_NUMBER() OVER (
      PARTITION BY product_id 
      ORDER BY individual_prediction DESC
    ) as prediction_rank,

    -- Calculate percentiles for large arrays
    PERCENT_RANK() OVER (
      PARTITION BY category 
      ORDER BY individual_prediction
    ) as category_prediction_percentile,

    -- Moving averages for time series analysis
    AVG(individual_prediction) OVER (
      PARTITION BY product_id 
      ORDER BY analysis_timestamp 
      ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING
    ) as smoothed_prediction

  FROM prediction_expanded
),

-- Aggregate large array results  
category_prediction_summary AS (
  SELECT 
    category,
    scenario_name,

    -- Prediction array aggregations
    COUNT(*) as total_predictions,
    ROUND(AVG(individual_prediction), 2) as avg_prediction,
    ROUND(STDDEV(individual_prediction), 2) as prediction_volatility,
    ROUND(MIN(individual_prediction), 2) as min_prediction,
    ROUND(MAX(individual_prediction), 2) as max_prediction,

    -- Percentile analysis of large arrays
    ROUND(PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY individual_prediction), 2) as p25_prediction,
    ROUND(PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY individual_prediction), 2) as median_prediction,
    ROUND(PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY individual_prediction), 2) as p75_prediction,
    ROUND(PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY individual_prediction), 2) as p95_prediction,

    -- Market intelligence aggregations
    ROUND(AVG(supplier_rating), 2) as avg_supplier_rating,
    ROUND(AVG(quality_score), 2) as avg_quality_score,
    ROUND(AVG(inventory_turnover), 2) as avg_inventory_turnover,

    -- Product mix analysis
    COUNT(CASE WHEN product_tier = 'premium' THEN 1 END) as premium_predictions,
    COUNT(CASE WHEN product_tier = 'standard' THEN 1 END) as standard_predictions,

    -- Processing metadata
    FIRST(batch_id) as batch_id,
    CURRENT_TIMESTAMP() as gold_processing_timestamp

  FROM prediction_rankings
  GROUP BY category, scenario_name
)

-- Final aggregated results from large arrays
SELECT 
  category,
  scenario_name,
  total_predictions,
  avg_prediction,
  prediction_volatility,
  median_prediction,
  p95_prediction,

  -- Risk assessment based on array analysis
  CASE 
    WHEN prediction_volatility > avg_prediction * 0.5 THEN 'HIGH_RISK'
    WHEN prediction_volatility > avg_prediction * 0.25 THEN 'MEDIUM_RISK'
    ELSE 'LOW_RISK'
  END as volatility_risk_category,

  -- Market position based on aggregated arrays
  CASE 
    WHEN avg_supplier_rating >= 4.0 AND avg_quality_score >= 8.0 THEN 'MARKET_LEADER'
    WHEN avg_supplier_rating >= 3.0 AND avg_quality_score >= 6.0 THEN 'COMPETITIVE'
    ELSE 'DEVELOPING'
  END as market_position,

  premium_predictions,
  standard_predictions,
  batch_id,
  gold_processing_timestamp

FROM category_prediction_summary
ORDER BY category, scenario_name;

-- Create large array feature importance table
CREATE OR REPLACE TABLE databricks_clar.gold.feature_importance AS
WITH feature_expanded AS (
  SELECT 
    product_id,
    category,
    scenario_name,
    EXPLODE(enrichment_data.detailed_features) as feature_detail
  FROM databricks_clar.temp.large_product_analysis
),

feature_aggregated AS (
  SELECT 
    feature_detail.name as feature_name,
    feature_detail.importance as importance_level,
    category,

    -- Feature array statistics
    COUNT(*) as feature_occurrences,
    ROUND(AVG(feature_detail.score), 2) as avg_feature_score,
    ROUND(STDDEV(feature_detail.score), 2) as feature_score_volatility,
    ROUND(MIN(feature_detail.score), 2) as min_feature_score,
    ROUND(MAX(feature_detail.score), 2) as max_feature_score,

    -- Feature ranking within category
    RANK() OVER (PARTITION BY category ORDER BY AVG(feature_detail.score) DESC) as category_feature_rank,

    CURRENT_TIMESTAMP() as processed_timestamp

  FROM feature_expanded
  GROUP BY feature_detail.name, feature_detail.importance, category
)

SELECT 
  feature_name,
  importance_level,
  category,
  feature_occurrences,
  avg_feature_score,
  feature_score_volatility,
  category_feature_rank,

  -- Feature significance classification
  CASE 
    WHEN avg_feature_score >= 80 AND importance_level = 'high' THEN 'CRITICAL'
    WHEN avg_feature_score >= 60 AND importance_level IN ('high', 'medium') THEN 'IMPORTANT'
    WHEN avg_feature_score >= 40 THEN 'MODERATE'
    ELSE 'LOW_IMPACT'
  END as feature_significance,

  processed_timestamp

FROM feature_aggregated
ORDER BY category, category_feature_rank;

-- Output processing summary
SELECT 
  'Large array SQL processing complete' as status,
  (SELECT COUNT(*) FROM databricks_clar.gold.prediction_analytics) as prediction_records,
  (SELECT COUNT(*) FROM databricks_clar.gold.feature_importance) as feature_records,
  (SELECT COUNT(DISTINCT batch_id) FROM databricks_clar.gold.prediction_analytics) as batches_processed;

🚨 Exam Key Points

SQL Large Array Patterns:

  1. SQL → Table → SQL tasks create result tables, not JSON returns
  2. Table → Notebook → Read large arrays directly from tables
  3. Table → SQL → Process arrays with SQL functions (EXPLODE, AGGREGATE, etc.)
  4. Array Functions → Use EXPLODE(), TRANSFORM(), AGGREGATE() for large array processing

SQL Array Processing Functions:

-- Array explosion for row-per-element processing
EXPLODE(array_column) as element

-- Array aggregation without explosion  
ARRAY_MAX(array_column)
ARRAY_MIN(array_column)  
SIZE(array_column)

-- Complex array operations
TRANSFORM(array_column, x -> x * 1.2)
FILTER(array_column, x -> x > 100)
AGGREGATE(array_column, 0, (acc, x) -> acc + x)

Performance Benefits:

  • No JSON limits → Arrays stay in Delta format
  • SQL optimization → Catalyst optimizer handles large arrays
  • Columnar efficiency → Parquet array compression
  • Parallel processing → Distributed array operations

Common Exam Scenarios:

  • "SQL task with large result arrays" → Create result table, not JSON
  • "Process prediction arrays from SQL" → Use EXPLODE + GROUP BY
  • "Large feature vectors between SQL tasks" → Table-to-table flow
  • "Array aggregation across scenarios" → SQL array functions

The Pattern: SQL Creates Tables → Notebooks/SQL Read Tables → No JSON Size Limits

This enables unlimited array processing in SQL-heavy pipelines while maintaining performance and simplicity! 🫡
