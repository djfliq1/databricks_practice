The Cortana Overview

Large data arrays can break notebook task value passing due to JSON size limits and performance issues. The key is knowing when to pass arrays directly vs. when to use alternative patterns like temporary tables, volumes, or streaming approaches.

🚨 The Problem: Large Array Limitations

Size Limits and Performance Issues

# notebooks/01_large_array_problem.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Large Array Problem Demonstration
# MAGIC **When direct JSON passing breaks down**

# COMMAND ----------
# Example: Your products dataset scaled up
large_product_analysis = spark.sql("""
SELECT 
  product_id,
  product_name,
  category,
  price,
  ROUND(price * RAND(), 2) as predicted_demand,
  ROUND(price * (1 + RAND() * 0.3), 2) as competitor_price,
  DATE_ADD(current_date(), CAST(RAND() * 365 AS INT)) as next_review_date,

  -- Generate large nested object per product
  STRUCT(
    category as main_category,
    CASE WHEN price > 250 THEN 'premium' ELSE 'standard' END as tier,
    ARRAY(
      STRUCT('feature_1' as name, RAND() * 100 as score),
      STRUCT('feature_2' as name, RAND() * 100 as score),
      STRUCT('feature_3' as name, RAND() * 100 as score),
      STRUCT('feature_4' as name, RAND() * 100 as score),
      STRUCT('feature_5' as name, RAND() * 100 as score)
    ) as features,
    MAP(
      'supplier_rating', RAND() * 5,
      'quality_score', RAND() * 10,
      'inventory_turnover', RAND() * 50,
      'seasonal_factor', RAND() * 2
    ) as metrics
  ) as enrichment_data

FROM databricks_clar.bronze.products_bronze
""")

# Collect large complex data (THIS WILL CAUSE ISSUES)
large_array = large_product_analysis.collect()
print(f"🚨 Large array size: {len(large_array)} complex objects")

# COMMAND ----------
# 🚨 PROBLEM: Convert to JSON for task output
import json
from datetime import datetime

# This creates massive JSON that breaks task value limits
try:
    # Convert Spark rows to serializable dictionaries
    serializable_array = []
    for row in large_array[:5]:  # Just first 5 for demo
        row_dict = {
            "product_id": int(row["product_id"]),
            "product_name": str(row["product_name"]),
            "category": str(row["category"]),
            "price": float(row["price"]),
            "predicted_demand": float(row["predicted_demand"]),
            "competitor_price": float(row["competitor_price"]),
            "next_review_date": row["next_review_date"].isoformat(),

            # Large nested structure
            "enrichment": {
                "main_category": row["enrichment_data"]["main_category"],
                "tier": row["enrichment_data"]["tier"],
                "features": [
                    {"name": f["name"], "score": float(f["score"])} 
                    for f in row["enrichment_data"]["features"]
                ],
                "metrics": dict(row["enrichment_data"]["metrics"])
            }
        }
        serializable_array.append(row_dict)

    # Calculate JSON size
    json_output = json.dumps(serializable_array)
    json_size_mb = len(json_output.encode('utf-8')) / 1024 / 1024

    print(f"📊 JSON size for {len(serializable_array)} products: {json_size_mb:.2f} MB")
    print(f"🔥 Estimated size for all {len(large_array)} products: {json_size_mb * len(large_array) / len(serializable_array):.2f} MB")

    # 🚨 This would fail for large datasets
    if json_size_mb > 10:  # Simulate size limit
        raise ValueError(f"JSON too large: {json_size_mb:.2f} MB exceeds 10 MB limit")

except Exception as e:
    print(f"❌ Large array JSON serialization failed: {e}")

# COMMAND ----------
# Return error indication instead of large array
error_output = {
    "status": "LARGE_ARRAY_ERROR",
    "product_count": len(large_array),
    "estimated_json_size_mb": json_size_mb * len(large_array) / len(serializable_array),
    "error_message": "Array too large for direct JSON passing",
    "alternative_needed": True
}

dbutils.notebook.exit(json.dumps(error_output))

✅ Solution 1: Temporary Tables for Large Arrays

Producer Notebook: Write to Temp Table

# notebooks/02_large_array_temp_table.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Solution 1: Temporary Tables for Large Arrays
# MAGIC **Write large arrays to Delta tables instead of JSON**

# COMMAND ----------
# Generate the same large product analysis
large_product_analysis = spark.sql("""
SELECT 
  product_id,
  product_name,
  category,
  price,
  ROUND(price * RAND(), 2) as predicted_demand,
  ROUND(price * (1 + RAND() * 0.3), 2) as competitor_price,
  DATE_ADD(current_date(), CAST(RAND() * 365 AS INT)) as next_review_date,

  -- Flattened structure for better performance
  category as main_category,
  CASE WHEN price > 250 THEN 'premium' ELSE 'standard' END as tier,
  RAND() * 100 as feature_1_score,
  RAND() * 100 as feature_2_score,
  RAND() * 100 as feature_3_score,
  RAND() * 100 as feature_4_score,
  RAND() * 100 as feature_5_score,
  RAND() * 5 as supplier_rating,
  RAND() * 10 as quality_score,
  RAND() * 50 as inventory_turnover,
  RAND() * 2 as seasonal_factor,

  -- Processing metadata
  current_timestamp() as analysis_timestamp,
  uuid() as batch_id

FROM databricks_clar.bronze.products_bronze
""")

# COMMAND ----------
# 🎯 SOLUTION: Write to temporary table instead of JSON
temp_table_name = f"databricks_clar.temp.large_product_analysis_{spark.sql('SELECT uuid()').collect()[0][0].replace('-', '_')}"

large_product_analysis.write \
    .format("delta") \
    .mode("overwrite") \
    .saveAsTable(temp_table_name)

total_records = large_product_analysis.count()
batch_id = large_product_analysis.select("batch_id").first()["batch_id"]

print(f"✅ Wrote {total_records} records to {temp_table_name}")

# COMMAND ----------
# Calculate summary metrics for small JSON output
summary_metrics = spark.sql(f"""
SELECT 
  COUNT(*) as total_products,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(AVG(predicted_demand), 2) as avg_predicted_demand,
  COUNT(DISTINCT category) as categories,
  COUNT(CASE WHEN tier = 'premium' THEN 1 END) as premium_count,
  MIN(next_review_date) as earliest_review,
  MAX(next_review_date) as latest_review,
  ROUND(AVG(supplier_rating), 2) as avg_supplier_rating,
  '{batch_id}' as batch_id
FROM {temp_table_name}
""").collect()[0]

# COMMAND ----------
# 🔥 EXAM PATTERN: Return table reference + summary (small JSON)
import json

task_output = {
    # Instead of large array, return table reference
    "large_data_table": temp_table_name,
    "data_location": "temp_table",
    "batch_id": batch_id,

    # Small summary metrics that fit in JSON
    "summary": {
        "total_records": int(summary_metrics["total_products"]),
        "avg_price": float(summary_metrics["avg_price"]),
        "avg_predicted_demand": float(summary_metrics["avg_predicted_demand"]),
        "categories": int(summary_metrics["categories"]),
        "premium_count": int(summary_metrics["premium_count"]),
        "avg_supplier_rating": float(summary_metrics["avg_supplier_rating"])
    },

    # Processing metadata
    "processing_timestamp": spark.sql("SELECT current_timestamp()").collect()[0][0].isoformat(),
    "records_available": True,
    "data_format": "delta_table"
}

print(f"📊 Summary: {task_output['summary']['total_records']} records, avg price ${task_output['summary']['avg_price']}")
print(f"📍 Data location: {temp_table_name}")

dbutils.notebook.exit(json.dumps(task_output))

Consumer Notebook: Read from Temp Table

# notebooks/03_consume_temp_table.py

# COMMAND ----------
# MAGIC %md  
# MAGIC ## Consumer: Reading Large Arrays from Temp Tables
# MAGIC **Access large data via table reference instead of JSON**

# COMMAND ----------
import json

# Get upstream task output (small JSON with table reference)
producer_output = json.loads(dbutils.widgets.get("producer_output"))

large_data_table = producer_output["large_data_table"]
batch_id = producer_output["batch_id"]
summary = producer_output["summary"]
total_records = summary["total_records"]

print(f"📊 Consuming {total_records} records from {large_data_table}")
print(f"🔍 Batch ID: {batch_id}")

# COMMAND ----------
# 🎯 SOLUTION: Read large data directly from table (no JSON limits)
df_large_data = spark.table(large_data_table)

# Verify data integrity
actual_count = df_large_data.count()
actual_batch_id = df_large_data.select("batch_id").first()["batch_id"]

assert actual_count == total_records, f"Count mismatch: {actual_count} != {total_records}"
assert actual_batch_id == batch_id, f"Batch ID mismatch: {actual_batch_id} != {batch_id}"

print(f"✅ Successfully loaded {actual_count} records")

# COMMAND ----------
# Process large data with full access (no JSON serialization needed)
# Example: Category-specific processing of large dataset

categories = df_large_data.select("category").distinct().collect()
processing_results = {}

for category_row in categories:
    category = category_row["category"]

    # Filter large dataset for this category
    df_category = df_large_data.filter(col("category") == category)
    category_count = df_category.count()

    # Complex processing that would break JSON limits
    category_features = df_category.select(
        "product_id",
        "product_name", 
        "price",
        "predicted_demand",
        "feature_1_score",
        "feature_2_score",
        "feature_3_score",
        "feature_4_score", 
        "feature_5_score"
    ).collect()  # This is OK because we're processing by category

    # Calculate category-specific metrics
    avg_features = {
        "feature_1": sum(row["feature_1_score"] for row in category_features) / len(category_features),
        "feature_2": sum(row["feature_2_score"] for row in category_features) / len(category_features),
        "feature_3": sum(row["feature_3_score"] for row in category_features) / len(category_features),
        "feature_4": sum(row["feature_4_score"] for row in category_features) / len(category_features),
        "feature_5": sum(row["feature_5_score"] for row in category_features) / len(category_features)
    }

    # Create enhanced Silver table for this category
    df_enhanced = df_category.select(
        "*",
        lit(avg_features["feature_1"]).alias("category_avg_feature_1"),
        lit(avg_features["feature_2"]).alias("category_avg_feature_2"), 
        lit(avg_features["feature_3"]).alias("category_avg_feature_3"),
        lit(avg_features["feature_4"]).alias("category_avg_feature_4"),
        lit(avg_features["feature_5"]).alias("category_avg_feature_5"),

        # Feature rankings within category
        row_number().over(
            Window.partitionBy("category").orderBy(desc("feature_1_score"))
        ).alias("feature_1_rank"),

        current_timestamp().alias("silver_processing_timestamp")
    )

    # Write category-specific Silver table
    silver_table = f"databricks_clar.silver.products_enhanced_{category.lower()}"
    df_enhanced.write.format("delta").mode("overwrite").saveAsTable(silver_table)

    processing_results[category] = {
        "products_processed": category_count,
        "silver_table": silver_table,
        "avg_features": avg_features
    }

    print(f"✅ Processed {category}: {category_count} products → {silver_table}")

# COMMAND ----------
# Clean up temporary table
spark.sql(f"DROP TABLE IF EXISTS {large_data_table}")
print(f"🗑️  Cleaned up temporary table: {large_data_table}")

# COMMAND ----------
# Return processing summary (small JSON)
consumer_output = {
    "processing_complete": True,
    "categories_processed": len(processing_results),
    "total_products_processed": sum(r["products_processed"] for r in processing_results.values()),
    "silver_tables_created": [r["silver_table"] for r in processing_results.values()],
    "batch_id": batch_id,
    "temp_table_cleaned": True,
    "processing_results": processing_results  # This is manageable size
}

dbutils.notebook.exit(json.dumps(consumer_output))

✅ Solution 2: Volume Files for Large Arrays

Producer: Write Large Array to Volume

# notebooks/04_large_array_volume.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Solution 2: Volume Files for Large Arrays
# MAGIC **Write large arrays to Unity Catalog volumes as Parquet/JSON files**

# COMMAND ----------
import json
import uuid

# Generate large dataset
large_dataset = spark.sql("""
SELECT 
  product_id,
  product_name,
  category,
  price,
  -- Simulate large feature vectors
  ARRAY(RAND(), RAND(), RAND(), RAND(), RAND(), RAND(), RAND(), RAND(), RAND(), RAND()) as feature_vector,
  MAP('metric_1', RAND() * 100, 'metric_2', RAND() * 200, 'metric_3', RAND() * 50) as metrics,
  current_timestamp() as created_timestamp
FROM databricks_clar.bronze.products_bronze
""")

# COMMAND ----------
# 🎯 SOLUTION: Write to Unity Catalog volume as Parquet
batch_id = str(uuid.uuid4())
volume_path = f"/Volumes/databricks_clar/temp/large_arrays/batch_{batch_id}"

# Write as Parquet (efficient for large datasets)
large_dataset.write \
    .format("parquet") \
    .mode("overwrite") \
    .save(volume_path)

# Also write metadata as JSON
metadata = {
    "batch_id": batch_id,
    "record_count": large_dataset.count(),
    "created_timestamp": spark.sql("SELECT current_timestamp()").collect()[0][0].isoformat(),
    "schema": large_dataset.schema.json(),
    "file_format": "parquet"
}

metadata_path = f"{volume_path}_metadata.json"
dbutils.fs.put(metadata_path, json.dumps(metadata, indent=2))

print(f"✅ Wrote large dataset to: {volume_path}")
print(f"📋 Metadata written to: {metadata_path}")

# COMMAND ----------
# Return small reference to large data
task_output = {
    "data_location": "volume_file",
    "volume_path": volume_path,
    "metadata_path": metadata_path,
    "batch_id": batch_id,
    "record_count": metadata["record_count"],
    "file_format": "parquet",
    "access_pattern": "spark_read_parquet"
}

dbutils.notebook.exit(json.dumps(task_output))

Consumer: Read from Volume

# notebooks/05_consume_volume.py

# COMMAND ----------
import json

# Get volume file reference
producer_output = json.loads(dbutils.widgets.get("producer_output"))
volume_path = producer_output["volume_path"]
metadata_path = producer_output["metadata_path"]
batch_id = producer_output["batch_id"]

print(f"📂 Reading large data from: {volume_path}")

# COMMAND ----------
# Read metadata first
metadata_json = dbutils.fs.head(metadata_path)
metadata = json.loads(metadata_json)

expected_count = metadata["record_count"]
file_format = metadata["file_format"]

# COMMAND ----------
# 🎯 SOLUTION: Read large dataset from volume
if file_format == "parquet":
    df_large = spark.read.parquet(volume_path)
else:
    df_large = spark.read.json(volume_path)

actual_count = df_large.count()
assert actual_count == expected_count, f"Count mismatch: {actual_count} != {expected_count}"

print(f"✅ Successfully loaded {actual_count} records from volume")

# COMMAND ----------
# Process large dataset
# Example: Feature analysis that requires full dataset access
feature_analysis = df_large.select(
    "category",
    explode(col("feature_vector")).alias("feature_value")
).groupBy("category").agg(
    avg("feature_value").alias("avg_feature"),
    min("feature_value").alias("min_feature"),
    max("feature_value").alias("max_feature"),
    count("feature_value").alias("feature_count")
)

feature_results = feature_analysis.collect()

# Clean up volume files
dbutils.fs.rm(volume_path, True)
dbutils.fs.rm(metadata_path)

print(f"🗑️  Cleaned up volume files")

# COMMAND ----------
# Return analysis results
consumer_output = {
    "processing_complete": True,
    "records_processed": actual_count,
    "feature_analysis": [
        {
            "category": row["category"],
            "avg_feature": float(row["avg_feature"]),
            "min_feature": float(row["min_feature"]),
            "max_feature": float(row["max_feature"]),
            "feature_count": int(row["feature_count"])
        }
        for row in feature_results
    ],
    "batch_id": batch_id,
    "volume_files_cleaned": True
}

dbutils.notebook.exit(json.dumps(consumer_output))

✅ Solution 3: Chunked Processing for Very Large Arrays

Producer: Chunk Large Arrays

# notebooks/06_chunked_processing.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Solution 3: Chunked Processing for Very Large Arrays
# MAGIC **Break large arrays into manageable chunks**

# COMMAND ----------
import json

# Simulate very large dataset analysis
very_large_analysis = spark.sql("""
SELECT 
  product_id,
  category,
  price,
  STRUCT(
    RAND() as score_1,
    RAND() as score_2,
    RAND() as score_3,
    RAND() as score_4,
    RAND() as score_5
  ) as analysis_scores
FROM databricks_clar.bronze.products_bronze
CROSS JOIN (
  SELECT EXPLODE(SEQUENCE(1, 20)) as multiplier  -- Simulate 20x data expansion
)
""")

total_records = very_large_analysis.count()
print(f"📊 Very large dataset: {total_records} records")

# COMMAND ----------
# 🎯 SOLUTION: Create chunks with row numbers
chunk_size = 50  # Records per chunk
num_chunks = (total_records + chunk_size - 1) // chunk_size

chunked_data = very_large_analysis.select(
    "*",
    (row_number().over(Window.orderBy("product_id")) - 1).alias("row_num")
).withColumn(
    "chunk_id", 
    (col("row_num") / chunk_size).cast("int")
)

# COMMAND ----------
# Write each chunk as separate table
chunk_tables = []
chunk_summaries = []

for chunk_id in range(num_chunks):
    df_chunk = chunked_data.filter(col("chunk_id") == chunk_id)
    chunk_count = df_chunk.count()

    chunk_table = f"databricks_clar.temp.large_data_chunk_{chunk_id}"
    df_chunk.write.format("delta").mode("overwrite").saveAsTable(chunk_table)

    # Chunk summary
    chunk_summary = {
        "chunk_id": chunk_id,
        "chunk_table": chunk_table,
        "record_count": chunk_count,
        "avg_price": df_chunk.agg(avg("price")).collect()[0][0]
    }

    chunk_tables.append(chunk_table)
    chunk_summaries.append(chunk_summary)

    print(f"✅ Created chunk {chunk_id}: {chunk_count} records")

# COMMAND ----------
# Return chunk references (manageable JSON size)
task_output = {
    "data_location": "chunked_tables",
    "total_records": total_records,
    "chunk_size": chunk_size,
    "num_chunks": num_chunks,
    "chunk_tables": chunk_tables,
    "chunk_summaries": chunk_summaries,
    "processing_pattern": "for_each_chunk"
}

dbutils.notebook.exit(json.dumps(task_output))

🎯 Performance Comparison

Method JSON Size Performance Use Case
Direct JSON ❌ Large (>10MB) ❌ Slow serialization ❌ Don't use for >1000 complex records
Temp Tables ✅ Small metadata ✅ Fast Delta I/O ✅ Best for structured data processing
Volume Files ✅ Small reference ⚡ Fast Parquet I/O ✅ Best for file-based workflows
Chunked Processing ✅ Manageable chunks ⚡ Parallel processing ✅ Best for very large datasets

🚨 Exam Key Points

Large Array Anti-Patterns:

  • ❌ dbutils.notebook.exit(json.dumps(large_array)) → Exceeds size limits
  • ❌ Passing 1000+ complex objects via JSON → Performance issues
  • ❌ Collecting entire large DataFrames to driver → Memory issues

Recommended Patterns:

  • ✅ Temp Tables → df.write.saveAsTable() + table reference
  • ✅ Volume Files → df.write.parquet() + file path reference
  • ✅ Chunked Processing → Split + For Each + aggregate
  • ✅ Summary + Reference → Small JSON with data location

When to Use Each:

  • Small arrays (<100 records) → Direct JSON passing
  • Medium arrays (100-1000 records) → Temp tables
  • Large arrays (1000+ records) → Volume files or chunked processing
  • Very large arrays (10,000+ records) → Chunked with For Each

The Rule: Never serialize large arrays to JSON. Always use external storage (tables/files/volumes) with small reference metadata.

This transforms large data bottlenecks into efficient data pipelines — essential for production-scale data engineering! 🫡
