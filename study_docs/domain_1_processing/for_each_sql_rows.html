The Cortana Overview

For Each tasks iterate over lists of values, executing the same logic for each item. Combined with SQL row results, this creates per-category processing, per-customer pipelines, and parallel execution patterns. Essential for scalable data processing where each row needs individual handling.

🎯 Core Pattern: SQL Rows → List → For Each Task

Step 1: SQL Analysis to Generate Row List

# notebooks/01_category_analysis_for_each.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Category Analysis - Generate List for For Each Processing
# MAGIC **SQL rows converted to list for parallel category processing**

# COMMAND ----------
# 🎯 SQL query to get category breakdown (multiple rows)
category_analysis = spark.sql("""
SELECT 
  category,
  COUNT(*) as product_count,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(MIN(price), 2) as min_price,
  ROUND(MAX(price), 2) as max_price,
  ROUND(SUM(price), 2) as total_value,

  -- Processing priority based on value
  CASE 
    WHEN AVG(price) > 250 THEN 'HIGH'
    WHEN AVG(price) > 150 THEN 'MEDIUM'
    ELSE 'LOW'
  END as priority,

  -- Processing complexity
  CASE 
    WHEN COUNT(*) > 25 THEN 'COMPLEX'
    WHEN COUNT(*) > 15 THEN 'STANDARD'
    ELSE 'SIMPLE'
  END as complexity,

  -- Estimated processing time (minutes)
  CASE 
    WHEN COUNT(*) > 25 AND AVG(price) > 250 THEN 15
    WHEN COUNT(*) > 15 THEN 10
    ELSE 5
  END as estimated_minutes

FROM databricks_clar.bronze.products_bronze
GROUP BY category
HAVING COUNT(*) >= 1  -- Only categories with products
ORDER BY avg_price DESC
""")

# Collect all category rows for For Each processing
category_rows = category_analysis.collect()

# COMMAND ----------
# 🔥 EXAM FOCUS: Convert SQL rows to For Each parameter list

# Method 1: Simple list of category names
category_names = [row["category"] for row in category_rows]

# Method 2: List of category configurations (complex objects)
category_configs = []
for row in category_rows:
    config = {
        "category_name": row["category"],
        "product_count": int(row["product_count"]),
        "avg_price": float(row["avg_price"]),
        "min_price": float(row["min_price"]),
        "max_price": float(row["max_price"]),
        "total_value": float(row["total_value"]),
        "priority": row["priority"],
        "complexity": row["complexity"],
        "estimated_minutes": int(row["estimated_minutes"]),

        # Derived processing parameters
        "processing_mode": "premium" if row["priority"] == "HIGH" else "standard",
        "batch_size": 1000 if row["complexity"] == "SIMPLE" else 500,
        "enable_caching": row["product_count"] > 20
    }
    category_configs.append(config)

# COMMAND ----------
# 🎯 Your Products Dataset Example Results
print("📊 Category Analysis Results:")
for i, row in enumerate(category_rows):
    print(f"  {i+1}. {row['category']}: {row['product_count']} products, avg ${row['avg_price']} ({row['priority']} priority)")

# Expected output for your data:
# 1. Books: 22 products, avg $271.89 (HIGH priority)
# 2. Toys: 21 products, avg $261.87 (HIGH priority)  
# 3. Electronics: 14 products, avg $241.36 (MEDIUM priority)
# 4. Furniture: 18 products, avg $230.32 (MEDIUM priority)
# 5. Clothing: 20 products, avg $199.84 (MEDIUM priority)

# COMMAND ----------
# Return both simple list and complex configs for For Each tasks
import json

task_output = {
    "category_count": len(category_rows),
    "category_names": category_names,  # Simple list for basic For Each
    "category_configs": category_configs,  # Complex objects for advanced For Each
    "total_products": sum(row["product_count"] for row in category_rows),
    "high_priority_categories": [row["category"] for row in category_rows if row["priority"] == "HIGH"],
    "estimated_total_time": sum(row["estimated_minutes"] for row in category_rows)
}

print(f"✅ Generated {len(category_names)} categories for For Each processing")
print(f"🔥 High priority: {task_output['high_priority_categories']}")
print(f"⏱️  Total estimated time: {task_output['estimated_total_time']} minutes")

dbutils.notebook.exit(json.dumps(task_output))

📋 Single Category Processing (For Each Worker)

# notebooks/02_process_single_category.py
# This runs once per category (5 times for your dataset)

# COMMAND ----------
# MAGIC %md
# MAGIC ## Process Single Category - For Each Worker
# MAGIC **Executed once per category from SQL row analysis**

# COMMAND ----------
import json

# Get the current category being processed
category_name = dbutils.widgets.get("category_name")  # "Books", "Toys", etc.

print(f"🎯 Processing category: {category_name}")

# COMMAND ----------
# Load only this category's data
df_category = spark.sql(f"""
SELECT *
FROM databricks_clar.bronze.products_bronze 
WHERE category = '{category_name}'
""")

category_count = df_category.count()
category_avg_price = df_category.agg({"price": "avg"}).collect()[0][0]

print(f"📊 Found {category_count} products with avg price ${category_avg_price:.2f}")

# COMMAND ----------
# Category-specific processing logic
if category_avg_price > 250:
    # Premium processing for high-value categories
    df_processed = df_category.select(
        "*",
        lit("premium").alias("processing_tier"),
        (col("price") * 0.30).alias("estimated_margin"),
        when(col("price") > category_avg_price * 1.5, "luxury")
        .when(col("price") > category_avg_price, "premium")
        .otherwise("standard").alias("price_segment"),
        current_timestamp().alias("processed_timestamp")
    )
    processing_mode = "premium"

else:
    # Standard processing
    df_processed = df_category.select(
        "*",
        lit("standard").alias("processing_tier"),
        when(col("price") > category_avg_price, "above_avg")
        .otherwise("below_avg").alias("price_position"),
        current_timestamp().alias("processed_timestamp")
    )
    processing_mode = "standard"

# COMMAND ----------
# Write category-specific Silver table
silver_table = f"databricks_clar.silver.products_{category_name.lower()}"
df_processed.write.format("delta").mode("overwrite").saveAsTable(silver_table)

# Category-specific data quality check
null_count = df_processed.filter(col("product_id").isNull()).count()
quality_score = max(0, 100 - (null_count / category_count * 100)) if category_count > 0 else 0

print(f"✅ Created {silver_table} with {category_count} products")
print(f"📈 Processing mode: {processing_mode}, Quality: {quality_score:.1f}%")

# COMMAND ----------
# Return results for this category iteration
category_result = {
    "category": category_name,
    "products_processed": category_count,
    "avg_price": round(category_avg_price, 2),
    "processing_mode": processing_mode,
    "silver_table": silver_table,
    "quality_score": round(quality_score, 1),
    "null_count": null_count
}

dbutils.notebook.exit(json.dumps(category_result))

🔧 Advanced For Each with Complex SQL Objects

# notebooks/03_process_category_advanced.py
# Advanced For Each worker with full category configuration

# COMMAND ----------
import json

# Get the complex category configuration object
category_config_raw = dbutils.widgets.get("category_config")
category_config = json.loads(category_config_raw)

# Extract configuration parameters
category_name = category_config["category_name"]
product_count = category_config["product_count"]
avg_price = category_config["avg_price"]
priority = category_config["priority"]
complexity = category_config["complexity"]
processing_mode = category_config["processing_mode"]
batch_size = category_config["batch_size"]
enable_caching = category_config["enable_caching"]
estimated_minutes = category_config["estimated_minutes"]

print(f"🎯 Processing {category_name}: {priority} priority, {complexity} complexity")
print(f"⚙️  Mode: {processing_mode}, Batch: {batch_size}, Cache: {enable_caching}")

# COMMAND ----------
# Load category data with caching optimization
df_category = spark.sql(f"""
SELECT * FROM databricks_clar.bronze.products_bronze 
WHERE category = '{category_name}'
""")

# Apply caching if configured
if enable_caching:
    df_category = df_category.cache()
    print("💾 Applied caching for large category")

# COMMAND ----------
# 🔥 EXAM PATTERN: Adaptive processing based on SQL-derived parameters

if processing_mode == "premium":
    # Premium processing with advanced analytics
    df_processed = df_category.select(
        "*",
        lit(priority.lower()).alias("priority_tier"),
        lit(complexity.lower()).alias("complexity_tier"),

        # Advanced price analytics
        (col("price") / avg_price).alias("price_ratio"),
        when(col("price") > avg_price + (avg_price * 0.5), "luxury")
        .when(col("price") > avg_price + (avg_price * 0.2), "premium")
        .when(col("price") > avg_price - (avg_price * 0.2), "standard")
        .otherwise("budget").alias("detailed_segment"),

        # Category-specific margins
        (col("price") * 
         when(priority == "HIGH", 0.35)
         .when(priority == "MEDIUM", 0.25)
         .otherwise(0.15)
        ).alias("category_margin"),

        # Processing metadata
        lit(batch_size).alias("batch_size_used"),
        lit(estimated_minutes).alias("processing_time_estimate"),
        current_timestamp().alias("processed_timestamp")
    )

else:
    # Standard processing
    df_processed = df_category.select(
        "*",
        lit(priority.lower()).alias("priority_tier"),
        when(col("price") > avg_price, "above_average")
        .otherwise("below_average").alias("simple_segment"),
        current_timestamp().alias("processed_timestamp")
    )

# COMMAND ----------
# Category-specific transformations based on complexity
if complexity == "COMPLEX":
    # Additional enrichment for complex categories
    df_processed = df_processed.withColumn(
        "complexity_score", 
        when(col("price") > avg_price * 1.5, 5)
        .when(col("price") > avg_price * 1.2, 4)
        .when(col("price") > avg_price * 0.8, 3)
        .when(col("price") > avg_price * 0.5, 2)
        .otherwise(1)
    ).withColumn(
        "investment_category",
        when(col("complexity_score") >= 4, "high_investment")
        .when(col("complexity_score") >= 3, "medium_investment")
        .otherwise("low_investment")
    )

# COMMAND ----------
# Write with dynamic partitioning based on priority
silver_table = f"databricks_clar.silver.products_{category_name.lower()}_{priority.lower()}"

if priority == "HIGH":
    # High priority gets partitioned tables
    df_processed.write \
        .format("delta") \
        .mode("overwrite") \
        .partitionBy("detailed_segment") \
        .saveAsTable(silver_table)
    partitioning = "by_segment"
else:
    # Standard priority gets non-partitioned tables  
    df_processed.write \
        .format("delta") \
        .mode("overwrite") \
        .saveAsTable(silver_table)
    partitioning = "none"

# COMMAND ----------
# Advanced quality metrics based on configuration
processed_count = df_processed.count()
null_ids = df_processed.filter(col("product_id").isNull()).count()
invalid_prices = df_processed.filter(col("price") <= 0).count()

quality_score = max(0, 100 - ((null_ids + invalid_prices) / processed_count * 100)) if processed_count > 0 else 0

# Category performance vs prediction
processing_variance = abs(processed_count - product_count) / product_count if product_count > 0 else 0

print(f"✅ Completed {category_name}: {processed_count} products processed")
print(f"📊 Quality: {quality_score:.1f}%, Variance: {processing_variance:.1%}")

# COMMAND ----------
# Return comprehensive results for aggregation
advanced_result = {
    "category": category_name,
    "input_config": category_config,  # Pass through original config
    "processing_results": {
        "products_processed": processed_count,
        "processing_mode": processing_mode,
        "priority": priority,
        "complexity": complexity,
        "silver_table": silver_table,
        "partitioning": partitioning,
        "caching_used": enable_caching,
        "batch_size": batch_size
    },
    "quality_metrics": {
        "quality_score": round(quality_score, 1),
        "null_ids": null_ids,
        "invalid_prices": invalid_prices,
        "processing_variance": round(processing_variance * 100, 1)
    },
    "performance": {
        "estimated_minutes": estimated_minutes,
        "actual_count": processed_count,
        "expected_count": product_count
    }
}

dbutils.notebook.exit(json.dumps(advanced_result))

# notebooks/04_aggregate_category_results.py
# Collect and analyze all For Each results

# COMMAND ----------
import json

# Get the original category configurations
category_configs_raw = dbutils.widgets.get("category_configs")
category_configs = json.loads(category_configs_raw)

print(f"📊 Aggregating results from {len(category_configs)} category For Each iterations")

# COMMAND ----------
# 🔥 EXAM PATTERN: Read For Each results from system tables

# Query job run results to get For Each outputs
job_run_id = dbutils.widgets.get("job_run_id", spark.sql("SELECT uuid() as id").collect()[0]["id"])

# In real implementation, you'd query system tables:
# for_each_results = spark.sql(f"""
# SELECT task_key, task_values 
# FROM system.lakeflow.task_runs 
# WHERE run_id = '{job_run_id}' 
#   AND task_key LIKE 'process_category_advanced_%'
# """).collect()

# For demo, simulate the For Each results based on your data
simulated_results = []
for config in category_configs:
    category = config["category_name"]
    result = {
        "category": category,
        "processing_results": {
            "products_processed": config["product_count"],
            "processing_mode": config["processing_mode"],
            "priority": config["priority"],
            "silver_table": f"databricks_clar.silver.products_{category.lower()}_{config['priority'].lower()}"
        },
        "quality_metrics": {
            "quality_score": 98.5 if config["priority"] == "HIGH" else 96.2,
            "null_ids": 0,
            "invalid_prices": 0
        }
    }
    simulated_results.append(result)

# COMMAND ----------
# Aggregate For Each processing results
total_products_processed = sum(r["processing_results"]["products_processed"] for r in simulated_results)
high_priority_count = sum(1 for r in simulated_results if r["processing_results"]["priority"] == "HIGH")
premium_processing_count = sum(1 for r in simulated_results if r["processing_results"]["processing_mode"] == "premium")

avg_quality_score = sum(r["quality_metrics"]["quality_score"] for r in simulated_results) / len(simulated_results)

# Processing summary by priority
priority_summary = {}
for result in simulated_results:
    priority = result["processing_results"]["priority"]
    if priority not in priority_summary:
        priority_summary[priority] = {"count": 0, "products": 0}
    priority_summary[priority]["count"] += 1
    priority_summary[priority]["products"] += result["processing_results"]["products_processed"]

# COMMAND ----------
# Create aggregated Silver tables list for Gold processing
silver_tables_created = [r["processing_results"]["silver_table"] for r in simulated_results]

# Generate Gold aggregation strategy based on For Each results
gold_strategy = "priority_based" if high_priority_count > 0 else "standard"

print(f"✅ For Each Processing Complete:")
print(f"   📊 Total products: {total_products_processed}")
print(f"   🏆 High priority categories: {high_priority_count}")
print(f"   ⚡ Premium processing: {premium_processing_count}")
print(f"   📈 Average quality: {avg_quality_score:.1f}%")
print(f"   🎯 Gold strategy: {gold_strategy}")

# COMMAND ----------
# Return aggregated results for downstream Gold processing
aggregated_output = {
    "for_each_summary": {
        "categories_processed": len(simulated_results),
        "total_products": total_products_processed,
        "high_priority_categories": high_priority_count,
        "premium_processing_categories": premium_processing_count,
        "average_quality_score": round(avg_quality_score, 1)
    },
    "priority_breakdown": priority_summary,
    "silver_tables": silver_tables_created,
    "gold_strategy": gold_strategy,
    "category_results": simulated_results  # Full results for detailed analysis
}

dbutils.notebook.exit(json.dumps(aggregated_output))

🚨 Exam Key Points

For Each with SQL Rows Patterns:

  1. SQL Analysis → Multiple rows → Convert to list → For Each input
  2. Simple For Each → List of strings/IDs for basic iteration
  3. Complex For Each → List of configuration objects for advanced processing
  4. Worker Tasks → Process one item per iteration with parameters
  5. Result Aggregation → Collect all For Each outputs for downstream tasks

For Each Task Configuration:

for_each_task:
  inputs: "{{tasks.analysis.values.category_names}}"  # List from SQL rows
  task:
    task_key: worker_task
    notebook_task:
      notebook_path: ./worker.py
      base_parameters:
        item: "{{input}}"  # Current iteration item

Common Exam Scenarios:

  • "Process each category separately" → SQL GROUP BY → For Each categories
  • "Per-customer pipeline execution" → SQL customer list → For Each customers
  • "Parallel processing based on SQL analysis" → SQL rows → For Each objects
  • "Dynamic table creation per group" → For Each with group-specific logic

Gotchas:

  • ❌ For Each inputs must be JSON arrays: ["item1", "item2"] or [{obj1}, {obj2}]
  • ❌ Worker task parameters are strings, parse JSON for complex objects
  • ❌ For Each results aren't automatically aggregated, need explicit aggregation task
  • ✅ Use {{input}} to access current iteration item in worker task
  • ✅ SQL rows → Python list → JSON → For Each inputs is the standard flow
  • ✅ Return structured JSON from worker tasks for result aggregation

The Pattern: SQL Analysis → Row Collection → List Generation → For Each Execution → Result Aggregation

This turns SQL GROUP BY results into parallel processing pipelines — essential for scalable category-based, customer-based, or partition-based data processing! 🫡
