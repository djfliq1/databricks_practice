The Cortana Overview

When you need single-value metrics from SQL (like totals, averages, max values, or summary stats), capture the first row of the result set. This is perfect for pipeline decision-making, quality gates, and conditional routing based on aggregate metrics.

🔍 Pattern: .collect()[0] for Single Row Results

Bronze Summary Metrics (First Row Pattern)

# notebooks/01_bronze_summary_metrics.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Bronze Summary Metrics - First Row Pattern
# MAGIC **Capturing single-value SQL results for pipeline decisions**

# COMMAND ----------
# 🎯 Pattern 1: Overall summary metrics (single row)
summary_metrics = spark.sql("""
SELECT 
  COUNT(*) as total_products,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(MIN(price), 2) as min_price,
  ROUND(MAX(price), 2) as max_price,
  ROUND(SUM(price), 2) as total_value,
  COUNT(DISTINCT category) as category_count,

  -- Quality metrics
  COUNT(CASE WHEN product_id IS NULL THEN 1 END) as null_products,
  COUNT(CASE WHEN price <= 0 THEN 1 END) as invalid_prices,
  COUNT(CASE WHEN product_name IS NULL OR product_name = '' THEN 1 END) as missing_names,

  -- Business insights
  COUNT(CASE WHEN price > 300 THEN 1 END) as premium_products,
  ROUND(AVG(CASE WHEN price > 300 THEN price END), 2) as premium_avg_price,

  -- Data freshness
  MAX(processDate) as latest_process_date,
  current_timestamp() as analysis_timestamp

FROM databricks_clar.bronze.products_bronze
""")

# 🔥 EXAM FOCUS: Get first (and only) row from summary query
summary_row = summary_metrics.collect()[0]

# COMMAND ----------
# 🎯 Pattern 2: Category performance leader (first row from ordered results)
top_category = spark.sql("""
SELECT 
  category,
  COUNT(*) as product_count,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(SUM(price), 2) as total_value,
  ROUND(MAX(price), 2) as highest_price_product
FROM databricks_clar.bronze.products_bronze
GROUP BY category
ORDER BY avg_price DESC
LIMIT 1
""")

top_category_row = top_category.collect()[0]

# COMMAND ----------
# 🎯 Pattern 3: Quality assessment (single row with multiple conditions)
quality_assessment = spark.sql("""
SELECT 
  -- Calculate quality score
  CASE 
    WHEN COUNT(*) = 0 THEN 0
    ELSE ROUND(
      100.0 * (COUNT(*) - COUNT(CASE WHEN product_id IS NULL OR price <= 0 OR product_name IS NULL THEN 1 END)) / COUNT(*), 
      2
    )
  END as quality_score,

  -- Data completeness ratio
  ROUND(100.0 * COUNT(CASE WHEN product_id IS NOT NULL AND price > 0 AND product_name IS NOT NULL THEN 1 END) / COUNT(*), 2) as completeness_ratio,

  -- Price distribution assessment
  CASE 
    WHEN AVG(price) > 250 THEN 'HIGH_VALUE'
    WHEN AVG(price) > 150 THEN 'MEDIUM_VALUE' 
    ELSE 'LOW_VALUE'
  END as value_category,

  -- Volume assessment
  CASE 
    WHEN COUNT(*) >= 100 THEN 'SUFFICIENT'
    WHEN COUNT(*) >= 50 THEN 'MINIMAL'
    ELSE 'INSUFFICIENT'
  END as volume_status,

  -- Freshness check (assuming daily loads)
  CASE 
    WHEN MAX(processDate) >= CURRENT_DATE() THEN 'FRESH'
    WHEN MAX(processDate) >= CURRENT_DATE() - INTERVAL 1 DAY THEN 'ACCEPTABLE'
    ELSE 'STALE'
  END as data_freshness

FROM databricks_clar.bronze.products_bronze
""")

quality_row = quality_assessment.collect()[0]

# COMMAND ----------
# 🎯 Pattern 4: Customer summary metrics
customer_summary = spark.sql("""
SELECT 
  COUNT(*) as total_customers,
  COUNT(DISTINCT domain) as unique_domains,
  COUNT(CASE WHEN domain IN ('gmail.com', 'yahoo.com', 'hotmail.com') THEN 1 END) as personal_email_count,
  COUNT(CASE WHEN domain NOT IN ('gmail.com', 'yahoo.com', 'hotmail.com') THEN 1 END) as business_email_count,

  -- Geographic distribution
  COUNT(DISTINCT location) as unique_locations,

  -- Signup trend (most recent signup)
  MAX(signup_date) as latest_signup,
  MIN(signup_date) as earliest_signup,

  -- Email quality
  COUNT(CASE WHEN email NOT LIKE '%@%' OR email IS NULL THEN 1 END) as invalid_emails

FROM databricks_clar.bronze.customers_bronze
""")

customer_row = customer_summary.collect()[0]

# COMMAND ----------
# 🔥 EXAM PATTERN: Convert first row results to task output values

# Extract values from first row (accessing by column name)
products_total = int(summary_row["total_products"])
products_avg_price = float(summary_row["avg_price"]) 
products_max_price = float(summary_row["max_price"])
products_min_price = float(summary_row["min_price"])
category_count = int(summary_row["category_count"])
null_products = int(summary_row["null_products"])
premium_count = int(summary_row["premium_products"])

# Top category details (first row from ordered query)
leading_category = top_category_row["category"]
leading_avg_price = float(top_category_row["avg_price"])
leading_product_count = int(top_category_row["product_count"])

# Quality assessment (first row calculations)
quality_score = float(quality_row["quality_score"])
completeness_ratio = float(quality_row["completeness_ratio"])
value_category = quality_row["value_category"]  # String enum
volume_status = quality_row["volume_status"]    # String enum
data_freshness = quality_row["data_freshness"]  # String enum

# Customer metrics (first row aggregation)
customers_total = int(customer_row["total_customers"])
unique_domains = int(customer_row["unique_domains"]) 
business_customers = int(customer_row["business_email_count"])
personal_customers = int(customer_row["personal_email_count"])

# COMMAND ----------
# 🎯 Business logic based on first row SQL results

# Quality gate decisions
quality_passed = quality_score >= 95 and completeness_ratio >= 95
volume_sufficient = volume_status in ['SUFFICIENT', 'MINIMAL']
data_fresh = data_freshness in ['FRESH', 'ACCEPTABLE']
pipeline_ready = quality_passed and volume_sufficient and data_fresh

# Processing mode decisions
if value_category == 'HIGH_VALUE' and premium_count > 10:
    processing_mode = 'PREMIUM'
elif value_category == 'MEDIUM_VALUE':
    processing_mode = 'STANDARD' 
else:
    processing_mode = 'ECONOMY'

# Customer segmentation decision
customer_mix = 'BUSINESS_FOCUSED' if business_customers > personal_customers else 'CONSUMER_FOCUSED'

# COMMAND ----------
# 🔥 EXAM FOCUS: Single-value task output for downstream routing
import json
from datetime import datetime

task_output = {
    # Single values from first row queries
    "total_products": products_total,
    "avg_price": products_avg_price,
    "max_price": products_max_price, 
    "min_price": products_min_price,
    "category_count": category_count,
    "premium_product_count": premium_count,

    # Top performer (first row from ordered query)
    "leading_category": leading_category,
    "leading_avg_price": leading_avg_price,
    "leading_product_count": leading_product_count,

    # Quality metrics (calculated first row)
    "quality_score": quality_score,
    "completeness_ratio": completeness_ratio,
    "value_category": value_category,
    "volume_status": volume_status,
    "data_freshness": data_freshness,

    # Customer summary (first row aggregation)
    "total_customers": customers_total,
    "unique_domains": unique_domains,
    "business_customers": business_customers,
    "personal_customers": personal_customers,

    # Decision flags (derived from first row results)
    "quality_passed": quality_passed,
    "volume_sufficient": volume_sufficient,
    "data_fresh": data_fresh,
    "pipeline_ready": pipeline_ready,
    "processing_mode": processing_mode,
    "customer_mix": customer_mix,

    # Metadata
    "analysis_timestamp": datetime.now().isoformat(),

}

print(f"✅ Summary Analysis Complete:")
print(f"   📊 Products: {products_total} (avg: ${products_avg_price})")
print(f"   🏆 Top Category: {leading_category} (${leading_avg_price})")

🎯 Downstream Task Consuming First Row Results

# notebooks/02_silver_using_first_row.py

# COMMAND ----------
import json

# Access single values from upstream first row analysis
bronze_summary = json.loads(dbutils.widgets.get("bronze_summary"))

# Extract single-value metrics
avg_price = bronze_summary["avg_price"]
leading_category = bronze_summary["leading_category"]
processing_mode = bronze_summary["processing_mode"]
quality_score = bronze_summary["quality_score"]
customer_mix = bronze_summary["customer_mix"]

print(f"🎯 Processing with mode: {processing_mode}")
print(f"📊 Based on avg price: ${avg_price}, top category: {leading_category}")

# COMMAND ----------
# Adaptive Silver processing based on single-value decisions
df_products = spark.table("databricks_clar.bronze.products_bronze")

if processing_mode == 'PREMIUM':
    # Premium processing for high-value datasets
    df_silver = df_products.select(
        "*",
        lit("premium").alias("tier"),
        when(col("category") == leading_category, "flagship")
        .otherwise("standard").alias("category_priority"),
        (col("price") * 0.25).alias("margin_estimate"),
        when(col("price") > avg_price * 1.5, "luxury")
        .when(col("price") > avg_price, "premium")
        .otherwise("standard").alias("price_segment")
    )

elif processing_mode == 'STANDARD':
    # Standard processing
    df_silver = df_products.select(
        "*",
        lit("standard").alias("tier"),
        when(col("price") > avg_price, "above_average")
        .otherwise("below_average").alias("price_position")
    )

else:  # ECONOMY
    # Minimal processing for low-value datasets
    df_silver = df_products.select(
        "product_id",
        "product_name", 
        "category",
        "price",
        lit("economy").alias("tier")
    )

# COMMAND ----------
# Customer processing based on single-value customer mix
df_customers = spark.table("databricks_clar.bronze.customers_bronze")

if customer_mix == 'BUSINESS_FOCUSED':
    # Enhanced business customer processing
    df_customers_silver = df_customers.select(
        "*",
        when(col("domain").isin("gmail.com", "yahoo.com", "hotmail.com"), "personal")
        .otherwise("business").alias("customer_type"),
        lit("b2b_enhanced").alias("processing_tier")
    )
else:
    # Consumer-focused processing
    df_customers_silver = df_customers.select(
        "*",
        lit("consumer").alias("customer_type"),
        lit("b2c_standard").alias("processing_tier")
    )

# COMMAND ----------
# Write Silver tables with processing mode in name
silver_suffix = processing_mode.lower()
products_table = f"databricks_clar.silver.products_{silver_suffix}"
customers_table = f"databricks_clar.silver.customers_{customer_mix.lower()}"

df_silver.write.format("delta").mode("overwrite").saveAsTable(products_table)
df_customers_silver.write.format("delta").mode("overwrite").saveAsTable(customers_table)

# COMMAND ----------
# Return processing results
silver_output = {
    "processing_mode_used": processing_mode,
    "customer_mix_used": customer_mix,
    "products_table_created": products_table,
    "customers_table_created": customers_table,
    "input_quality_score": quality_score,
    "records_processed": df_silver.count() + df_customers_silver.count()
}

dbutils.notebook.exit(json.dumps(silver_output))

threshold_check = spark.sql("""

🚨 Exam Key Points

First Row Patterns:

  1. Summary Metrics → COUNT(*), AVG(), MAX() in single row
  2. Top/Bottom Analysis → ORDER BY ... LIMIT 1 for extremes
  3. Quality Assessment → Calculated scores in single row
  4. Business Logic → CASE WHEN conditions returning single values
  5. Boolean Flags → True/false decisions from single row calculations

Access Pattern:

# Always the same pattern
result = spark.sql("SELECT ...").collect()[0]
value = result["column_name"]  # Extract by column name

Common Exam Scenarios:

  • "Get the highest-performing category" → ORDER BY avg_price DESC LIMIT 1
  • "Quality gate based on completeness ratio" → Single row calculation
  • "Conditional processing based on data volume" → COUNT(*) first row
  • "Route based on value category" → CASE WHEN AVG(price)... first row

Gotchas:

  • ❌ result[0]["column"] → Use result["column"] after .collect()[0]
  • ❌ Forgetting .collect()[0] → Gets DataFrame, not row data
  • ❌ Multiple rows when expecting one → Use LIMIT 1 or aggregate functions
  • ✅ Always handle potential None values from SQL results
  • ✅ Convert to appropriate Python types (int(), float(), str())

The Pattern: Single SQL Query → First Row → Extract Values → Task Output → Downstream Decisions

This turns aggregate SQL analysis into single-value pipeline controls — perfect for quality gates, routing decisions, and adaptive processing! 🫡
