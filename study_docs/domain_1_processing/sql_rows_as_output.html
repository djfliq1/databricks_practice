The Cortana Overview

SQL tasks can't directly return structured data like notebook tasks, BUT you can capture SQL query results as task outputs using notebooks that execute SQL or SQL tasks that write to result tables for downstream consumption.

🎯 Method 1: Notebook with SQL Queries (Recommended)

Bronze Analysis Notebook (SQL + Python Output)

# notebooks/01_bronze_sql_analysis.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Bronze Analysis with SQL Row Results
# MAGIC **Capturing SQL query results as task output values**

# COMMAND ----------
# Your Bronze data analysis using SQL
bronze_products_stats = spark.sql("""
SELECT 
  category,
  COUNT(*) as product_count,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(MIN(price), 2) as min_price,
  ROUND(MAX(price), 2) as max_price,
  ROUND(SUM(price), 2) as total_value
FROM databricks_clar.bronze.products_bronze 
GROUP BY category
ORDER BY avg_price DESC
""")

# Convert to Python data structures for output
category_metrics = bronze_products_stats.collect()

# COMMAND ----------
# Process customer domain analysis
customer_domains = spark.sql("""
SELECT 
  domain,
  COUNT(*) as customer_count,
  COUNT(DISTINCT location) as unique_locations,
  MIN(signup_date) as first_signup,
  MAX(signup_date) as last_signup
FROM databricks_clar.bronze.customers_bronze
GROUP BY domain
HAVING COUNT(*) >= 2
ORDER BY customer_count DESC
LIMIT 10
""")

domain_metrics = customer_domains.collect()

# COMMAND ----------
# Quality assessment SQL
quality_check = spark.sql("""
SELECT 
  -- Product quality metrics
  (SELECT COUNT(*) FROM databricks_clar.bronze.products_bronze) as total_products,
  (SELECT COUNT(*) FROM databricks_clar.bronze.products_bronze WHERE product_id IS NULL) as null_products,
  (SELECT COUNT(*) FROM databricks_clar.bronze.products_bronze WHERE price <= 0) as invalid_prices,

  -- Customer quality metrics  
  (SELECT COUNT(*) FROM databricks_clar.bronze.customers_bronze) as total_customers,
  (SELECT COUNT(*) FROM databricks_clar.bronze.customers_bronze WHERE email IS NULL OR email NOT LIKE '%@%') as invalid_emails,

  -- Data freshness
  (SELECT MAX(bronze_ingestion_timestamp) FROM databricks_clar.bronze.products_bronze) as latest_product_load,
  (SELECT MAX(bronze_ingestion_timestamp) FROM databricks_clar.bronze.customers_bronze) as latest_customer_load
""")

quality_data = quality_check.collect()[0]

# COMMAND ----------
# 🎯 Convert SQL rows to structured output for downstream tasks

# Category breakdown for conditional processing
categories_output = []
top_category = None
lowest_category = None
high_value_categories = []

for row in category_metrics:
    category_dict = {
        "category": row["category"],
        "product_count": int(row["product_count"]), 
        "avg_price": float(row["avg_price"]),
        "min_price": float(row["min_price"]),
        "max_price": float(row["max_price"]),
        "total_value": float(row["total_value"])
    }
    categories_output.append(category_dict)

    # Identify categories for downstream routing
    if row["avg_price"] > 250:
        high_value_categories.append(row["category"])

    # Track extremes
    if top_category is None or row["avg_price"] > top_category["avg_price"]:
        top_category = category_dict
    if lowest_category is None or row["avg_price"] < lowest_category["avg_price"]:
        lowest_category = category_dict

# COMMAND ----------
# Domain analysis for customer segmentation
domains_output = []
business_domains = []
personal_domains = ['gmail.com', 'yahoo.com', 'hotmail.com', 'outlook.com']

for row in domain_metrics:
    domain_dict = {
        "domain": row["domain"],
        "customer_count": int(row["customer_count"]),
        "unique_locations": int(row["unique_locations"]),
        "first_signup": row["first_signup"].isoformat() if row["first_signup"] else None,
        "last_signup": row["last_signup"].isoformat() if row["last_signup"] else None,
        "is_business": row["domain"] not in personal_domains
    }
    domains_output.append(domain_dict)

    if domain_dict["is_business"]:
        business_domains.append(row["domain"])

# COMMAND ----------
# Quality metrics from SQL aggregation
quality_metrics = {
    "total_products": int(quality_data["total_products"]),
    "total_customers": int(quality_data["total_customers"]),
    "null_products": int(quality_data["null_products"]),
    "invalid_emails": int(quality_data["invalid_emails"]),
    "invalid_prices": int(quality_data["invalid_prices"]),
    "latest_product_load": quality_data["latest_product_load"].isoformat() if quality_data["latest_product_load"] else None,
    "latest_customer_load": quality_data["latest_customer_load"].isoformat() if quality_data["latest_customer_load"] else None,
}

# Calculate derived metrics
total_records = quality_metrics["total_products"] + quality_metrics["total_customers"]
total_issues = quality_metrics["null_products"] + quality_metrics["invalid_emails"] + quality_metrics["invalid_prices"]
quality_score = max(0, 100 - (total_issues / total_records * 100)) if total_records > 0 else 0

# COMMAND ----------
# 🔥 EXAM FOCUS: Complete task output with SQL row results
import json
from datetime import datetime

task_output = {
    # Aggregated metrics for quick access
    "summary": {
        "total_categories": len(categories_output),
        "total_products": quality_metrics["total_products"],
        "total_customers": quality_metrics["total_customers"],
        "quality_score": round(quality_score, 2),
        "high_value_category_count": len(high_value_categories)
    },

    # Detailed SQL row results
    "category_breakdown": categories_output,  # List of category dicts
    "domain_analysis": domains_output,        # List of domain dicts
    "quality_details": quality_metrics,      # Quality assessment

    # Key insights for downstream routing
    "top_performing_category": top_category,
    "lowest_performing_category": lowest_category, 
    "high_value_categories": high_value_categories,
    "business_domains": business_domains,

    # Decision flags
    "has_high_value_products": len(high_value_categories) > 0,
    "quality_passed": quality_score >= 95,
    "has_business_customers": len(business_domains) > 0,
    "data_freshness_ok": quality_data["latest_product_load"] is not None,

    # Processing metadata
    "analysis_timestamp": datetime.now().isoformat(),
    "total_issues_found": total_issues
}

print(f"✅ Analysis complete: {len(categories_output)} categories, Quality: {quality_score:.1f}%")
print(f"🎯 Top category: {top_category['category']} (${top_category['avg_price']})")
print(f"📊 High-value categories: {', '.join(high_value_categories) if high_value_categories else 'None'}")

# Return structured data for downstream tasks
dbutils.notebook.exit(json.dumps(task_output))

🎯 Method 2: SQL Task Writing to Result Tables

SQL Task that Creates Result Tables for Downstream Access

-- notebooks/02_bronze_analysis.sql  
-- SQL task that creates result tables instead of returning values directly

-- Create category analysis table
CREATE OR REPLACE TABLE databricks_clar.bronze.category_analysis AS
SELECT 
  'category_metrics' as analysis_type,
  category,
  COUNT(*) as product_count,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(MIN(price), 2) as min_price,
  ROUND(MAX(price), 2) as max_price,
  ROUND(SUM(price), 2) as total_value,
  CASE WHEN AVG(price) > 250 THEN true ELSE false END as is_high_value,
  current_timestamp() as analysis_timestamp
FROM databricks_clar.bronze.products_bronze 
GROUP BY category;

-- Create domain analysis table
CREATE OR REPLACE TABLE databricks_clar.bronze.domain_analysis AS
SELECT 
  'domain_metrics' as analysis_type,
  domain,
  COUNT(*) as customer_count,
  COUNT(DISTINCT location) as unique_locations,
  MIN(signup_date) as first_signup,
  MAX(signup_date) as last_signup,
  CASE 
    WHEN domain IN ('gmail.com', 'yahoo.com', 'hotmail.com') THEN 'Personal'
    ELSE 'Business' 
  END as domain_type,
  current_timestamp() as analysis_timestamp
FROM databricks_clar.bronze.customers_bronze
GROUP BY domain
HAVING COUNT(*) >= 1;

-- Create summary metrics table
CREATE OR REPLACE TABLE databricks_clar.bronze.pipeline_summary AS  
SELECT 
  'pipeline_metrics' as analysis_type,
  (SELECT COUNT(*) FROM databricks_clar.bronze.products_bronze) as total_products,
  (SELECT COUNT(*) FROM databricks_clar.bronze.customers_bronze) as total_customers,
  (SELECT COUNT(*) FROM databricks_clar.bronze.category_analysis WHERE is_high_value = true) as high_value_categories,
  (SELECT COUNT(*) FROM databricks_clar.bronze.domain_analysis WHERE domain_type = 'Business') as business_domains,
  (SELECT AVG(avg_price) FROM databricks_clar.bronze.category_analysis) as overall_avg_price,
  current_timestamp() as analysis_timestamp;

🔄 Method 3: Downstream Task Reading SQL Results

Silver Task that Consumes SQL Row Results

# notebooks/03_silver_consuming_sql_rows.py

# COMMAND ----------
import json

# 🎯 Method 1: Access structured JSON from upstream notebook task
bronze_analysis_raw = dbutils.widgets.get("bronze_analysis_output")  # {{tasks.bronze_analysis.values}}
bronze_data = json.loads(bronze_analysis_raw)

# Access specific SQL row results
categories = bronze_data["category_breakdown"]
domains = bronze_data["domain_analysis"] 
top_category = bronze_data["top_performing_category"]
high_value_categories = bronze_data["high_value_categories"]

print(f"📊 Processing {len(categories)} categories from SQL analysis")
print(f"🎯 Top performer: {top_category['category']} (${top_category['avg_price']})")

# COMMAND ----------
# 🎯 Method 2: Read from SQL result tables (if using SQL task approach)
if spark.catalog.tableExists("databricks_clar.bronze.category_analysis"):
    category_sql_results = spark.table("databricks_clar.bronze.category_analysis").collect()
    domain_sql_results = spark.table("databricks_clar.bronze.domain_analysis").collect()
    summary_sql_results = spark.table("databricks_clar.bronze.pipeline_summary").collect()[0]

    print(f"📋 Read {len(category_sql_results)} categories from SQL result table")

# COMMAND ----------
# 🔥 EXAM PATTERN: Adaptive processing based on SQL row results

# Process each category based on SQL analysis
for category in categories:
    category_name = category["category"]
    avg_price = category["avg_price"]
    product_count = category["product_count"]

    print(f"Processing {category_name}: {product_count} products, avg ${avg_price}")

    # Load category-specific data
    df_category = spark.sql(f"""
    SELECT * FROM databricks_clar.bronze.products_bronze 
    WHERE category = '{category_name}'
    """)

    # Adaptive enrichment based on SQL analysis
    if category_name in high_value_categories:
        # Premium processing for high-value categories
        df_enriched = df_category.select(
            "*",
            lit("premium").alias("processing_tier"),
            (col("price") * 0.25).alias("estimated_margin"),
            when(col("price") > avg_price * 1.5, "Luxury")
            .when(col("price") > avg_price, "Premium") 
            .otherwise("Standard").alias("price_segment")
        )
        print(f"  ✨ Applied premium processing for {category_name}")
    else:
        # Standard processing
        df_enriched = df_category.select(
            "*",
            lit("standard").alias("processing_tier"),
            when(col("price") > avg_price, "Above Average")
            .otherwise("Below Average").alias("price_position")
        )
        print(f"  📊 Applied standard processing for {category_name}")

    # Write category-specific Silver table
    silver_table = f"databricks_clar.silver.products_{category_name.lower().replace(' ', '_')}"
    df_enriched.write.format("delta").mode("overwrite").saveAsTable(silver_table)

# COMMAND ----------
# Process business vs personal customers based on domain analysis
business_customer_count = 0
personal_customer_count = 0

for domain in domains:
    domain_name = domain["domain"]
    customer_count = domain["customer_count"]
    is_business = domain["is_business"]

    if is_business:
        business_customer_count += customer_count
        # Enhanced processing for business customers
        spark.sql(f"""
        CREATE OR REPLACE TABLE databricks_clar.silver.customers_business AS
        SELECT 
          *,
          'business' as customer_segment,
          '{domain_name}' as primary_domain,
          'priority' as service_level
        FROM databricks_clar.bronze.customers_bronze 
        WHERE domain = '{domain_name}'
        """)
    else:
        personal_customer_count += customer_count

print(f"📊 Processed {business_customer_count} business + {personal_customer_count} personal customers")

# COMMAND ----------
# Return results based on SQL row analysis
silver_output = {
    "categories_processed": len(categories),
    "high_value_categories_count": len(high_value_categories),
    "business_customer_count": business_customer_count,
    "personal_customer_count": personal_customer_count,
    "top_category_processed": top_category["category"],
    "processing_approach": "adaptive_by_sql_analysis",
    "tables_created": [f"products_{cat['category'].lower().replace(' ', '_')}" for cat in categories]
}

dbutils.notebook.exit(json.dumps(silver_output))

            - task_key: bronze_sql_analysis
          sql_task:

              path: ./notebooks/02_bronze_analysis.sql

        # Consumer: Silver processing using SQL row results
        - task_key: silver_adaptive_processing

            - task_key: bronze_sql_analysis
            - task_key: sql_analysis_tables

🚨 Exam Key Points

SQL Rows as Output Patterns:

  1. Notebook + SQL → Best approach, returns structured JSON with row data
  2. SQL + Result Tables → SQL task writes tables, downstream reads them
  3. Hybrid Approach → SQL creates tables, notebook processes and returns structured output

Common Exam Scenarios:

  • "How do you pass query results between tasks?" → Notebook with SQL returning JSON
  • "SQL task output values" → SQL tasks don't return values directly, use result tables
  • "Conditional processing based on SQL aggregation" → SQL in notebook, return boolean flags
  • "Access specific rows in downstream task" → JSON array of row objects

Gotchas:

  • ❌ Pure SQL tasks cannot return task values directly
  • ❌ {{tasks.sql_task.values.column}} → SQL tasks don't produce values
  • ✅ {{tasks.notebook_with_sql.values.sql_results}} → Notebook can return SQL results
  • ✅ Use .collect() to convert Spark DataFrame rows to Python objects
  • ✅ Convert timestamps to .isoformat() for JSON serialization

Success Pattern:

SQL Analysis → Python Processing → JSON Output → Downstream Consumption

This turns your Bronze PySpark analysis into a data-driven orchestration system where SQL results control the flow and processing of downstream tasks. Essential for the 2026 exam! 🫡
