⏰ Schedules and Triggers in LakeFlow Jobs

Databricks Data Engineer Associate 2026 Certification

The Cortana Overview

Triggers determine WHEN your jobs run. Schedules are time-based triggers using cron expressions. But triggers also include file arrival, continuous streaming, external webhooks, and manual execution. Understanding trigger types and their use cases is critical for the 2026 exam.

🎯 Trigger Types Overview

Trigger Type When to Use Your Products Example
Scheduled Regular ETL, daily/weekly reports Process products CSV daily at 2 AM
File Arrival Event-driven processing New products file → trigger pipeline
Continuous Streaming data, real-time processing Continuous product updates stream
Manual Development, testing, ad-hoc runs Run product analysis on-demand
Webhook External system integration External catalog update → trigger job

schedules:
  daily_2am: "0 0 2 * * ?"           # Every day at 2 AM
  hourly: "0 0 * * * ?"              # Every hour at top of hour
  weekdays_6am: "0 0 6 ? * MON-FRI"  # Weekdays at 6 AM  
  weekly_sunday: "0 0 3 ? * SUN"     # Sundays at 3 AM
  monthly_1st: "0 0 1 1 * ?"         # 1st of month at 1 AM
  every_15min: "0 */15 * * * ?"      # Every 15 minutes
  twice_daily: "0 0 8,20 * * ?"      # 8 AM and 8 PM daily

File Arrival Processing Logic

# notebooks/02_file_arrival_processor.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## File Arrival Triggered Processing
# MAGIC **Process new files as they arrive in Unity Catalog volumes**

# COMMAND ----------
import json
from datetime import datetime
from pyspark.sql.functions import *

# Get file arrival context from job trigger
trigger_info = {
    "trigger_type": "file_arrival",
    "execution_time": datetime.now().isoformat(),
    "volume_path": "/Volumes/databricks_clar/bronze/incoming/products/"
}

print(f"🔔 File arrival triggered at {trigger_info['execution_time']}")

# COMMAND ----------
# 🔥 EXAM PATTERN: List and process new files
volume_path = "/Volumes/databricks_clar/bronze/incoming/products/"

# List files in the trigger directory
new_files = dbutils.fs.ls(volume_path)
csv_files = [f for f in new_files if f.name.endswith('.csv')]

print(f"📁 Found {len(csv_files)} CSV files to process:")
for file_info in csv_files:
    print(f"   - {file_info.name} ({file_info.size} bytes)")

# COMMAND ----------
# Process each new file
processed_files = []
total_records = 0

for file_info in csv_files:
    file_path = file_info.path
    file_name = file_info.name
    file_size = file_info.size

    print(f"🔄 Processing {file_name}...")

    try:
        # Load the new file
        df_new_products = spark.read.format("csv") \
            .option("header", True) \
            .option("inferSchema", True) \
            .load(file_path)

        # Add file metadata for tracking
        df_with_metadata = df_new_products.select(
            "*",
            lit(file_name).alias("source_file"),
            lit(file_size).alias("source_file_size"),
            current_timestamp().alias("ingestion_timestamp"),
            lit("file_arrival").alias("trigger_type")
        )

        file_record_count = df_with_metadata.count()
        total_records += file_record_count

        # Append to Bronze table (merge logic could be added)
        df_with_metadata.write \
            .format("delta") \
            .mode("append") \
            .saveAsTable("databricks_clar.bronze.products_bronze")

        # Move processed file to archive
        archive_path = f"/Volumes/databricks_clar/bronze/archive/products/{datetime.now().strftime('%Y/%m/%d')}/{file_name}"
        dbutils.fs.cp(file_path, archive_path)
        dbutils.fs.rm(file_path)  # Remove from incoming

        processed_files.append({
            "file_name": file_name,
            "file_size": file_size,
            "records_processed": file_record_count,
            "archive_path": archive_path,
            "processing_status": "SUCCESS"
        })

        print(f"✅ {file_name}: {file_record_count} records → archived")

    except Exception as e:
        print(f"❌ Error processing {file_name}: {e}")

        # Move failed file to error directory
        error_path = f"/Volumes/databricks_clar/bronze/errors/products/{file_name}"
        dbutils.fs.cp(file_path, error_path)
        dbutils.fs.rm(file_path)

        processed_files.append({
            "file_name": file_name,
            "file_size": file_size,
            "records_processed": 0,
            "error_path": error_path,
            "processing_status": "FAILED",
            "error_message": str(e)
        })

# COMMAND ----------
# Return file processing results
processing_output = {
    "trigger_type": "file_arrival",
    "processing_timestamp": datetime.now().isoformat(),
    "files_found": len(csv_files),
    "files_processed": len(processed_files),
    "total_records_ingested": total_records,
    "processed_files": processed_files,
    "volume_path": volume_path
}

print(f"🎯 File arrival processing complete:")
print(f"   📊 {len(csv_files)} files found, {len(processed_files)} processed")
print(f"   📈 {total_records} total records ingested")

dbutils.notebook.exit(json.dumps(processing_output))

Continuous Streaming Logic

# notebooks/03_continuous_streaming.py

# COMMAND ----------
# MAGIC %md
# MAGIC ## Continuous Product Streaming
# MAGIC **Continuously process product updates using Structured Streaming**

# COMMAND ----------
from pyspark.sql.functions import *
from pyspark.sql.streaming import DataStreamReader
import json

# COMMAND ----------
# 🎯 EXAM PATTERN: Structured Streaming with continuous trigger

# Set up streaming source (Auto Loader for files)
streaming_df = spark.readStream \
    .format("cloudFiles") \
    .option("cloudFiles.format", "csv") \
    .option("cloudFiles.schemaLocation", "/Volumes/databricks_clar/bronze/schemas/products/") \
    .option("header", "true") \
    .load("/Volumes/databricks_clar/bronze/streaming/products/")

# Add streaming metadata
streaming_with_metadata = streaming_df.select(
    "*",
    current_timestamp().alias("stream_ingestion_time"),
    input_file_name().alias("source_stream_file"),
    lit("continuous_stream").alias("trigger_type")
)

# COMMAND ----------
# Streaming transformations for your products data
transformed_stream = streaming_with_metadata.select(
    col("product_id"),
    col("product_name"),
    col("category"),
    col("price").cast("double"),
    col("processDate").cast("timestamp"),
    col("stream_ingestion_time"),
    col("source_stream_file"),
    col("trigger_type"),

    # Real-time enrichments
    when(col("price") > 250, "HIGH_VALUE")
    .when(col("price") > 100, "MEDIUM_VALUE")
    .otherwise("LOW_VALUE").alias("value_category"),

    # Streaming window for aggregations
    window(col("stream_ingestion_time"), "5 minutes").alias("processing_window")
)

# COMMAND ----------
# 🔥 CONTINUOUS TRIGGER: Write streaming results
streaming_query = transformed_stream.writeStream \
    .format("delta") \
    .outputMode("append") \
    .option("checkpointLocation", "/Volumes/databricks_clar/bronze/checkpoints/products_stream/") \
    .trigger(continuous="10 seconds") \
    .toTable("databricks_clar.bronze.products_streaming")

# For job context, we need to handle the streaming query
# In continuous jobs, the streaming query runs indefinitely
print(f"🔄 Continuous streaming started for products pipeline")
print(f"📊 Checkpoint location: /Volumes/databricks_clar/bronze/checkpoints/products_stream/")
print(f"⚡ Trigger interval: 10 seconds continuous")

# The streaming query continues running...
# In notebook context, you might want to run for a specific duration
streaming_query.awaitTermination(timeout=None)  # Runs indefinitely

Manual Execution Logic

# notebooks/04_manual_execution.py

# COMMAND ----------
# Manual execution with parameters
execution_mode = dbutils.widgets.get("execution_mode")
analysis_type = dbutils.widgets.get("analysis_type")
category_filter = dbutils.widgets.get("category_filter")
date_range = dbutils.widgets.get("date_range")

print(f"🎯 Manual execution started:")
print(f"   Mode: {execution_mode}")
print(f"   Analysis: {analysis_type}")
print(f"   Category: {category_filter}")
print(f"   Date range: {date_range}")

# COMMAND ----------
# Load data based on manual parameters
if category_filter == "all_categories":
    df_products = spark.table("databricks_clar.bronze.products_bronze")
else:
    df_products = spark.table("databricks_clar.bronze.products_bronze") \
        .filter(col("category") == category_filter)

# Apply date filtering for manual analysis
if date_range == "last_30_days":
    df_filtered = df_products.filter(
        col("processDate") >= date_sub(current_date(), 30)
    )
elif date_range == "last_7_days":
    df_filtered = df_products.filter(
        col("processDate") >= date_sub(current_date(), 7)
    )
else:
    df_filtered = df_products

record_count = df_filtered.count()
print(f"📊 Manual analysis dataset: {record_count} records")

# COMMAND ----------
# Perform analysis based on type
if analysis_type == "full_analysis":
    # Comprehensive analysis
    results = df_filtered.groupBy("category").agg(
        count("*").alias("product_count"),
        avg("price").alias("avg_price"),
        min("price").alias("min_price"),
        max("price").alias("max_price")
    ).collect()

elif analysis_type == "price_analysis":
    # Price-focused analysis
    results = df_filtered.select(
        col("category"),
        col("price")
    ).groupBy("category").agg(
        percentile_approx("price", 0.5).alias("median_price"),
        percentile_approx("price", 0.95).alias("p95_price")
    ).collect()

else:
    # Quick summary
    results = df_filtered.agg(
        count("*").alias("total_products"),
        countDistinct("category").alias("unique_categories")
    ).collect()

# Return manual execution results
manual_output = {
    "execution_mode": execution_mode,
    "analysis_type": analysis_type,
    "category_filter": category_filter,
    "date_range": date_range,
    "records_analyzed": record_count,
    "results": [row.asDict() for row in results],
    "manual_execution_complete": True
}

dbutils.notebook.exit(json.dumps(manual_output))

Webhook Processing

# notebooks/05_webhook_processor.py

# COMMAND ----------
import json
from datetime import datetime

# Webhook context (would be provided by external system)
webhook_context = {
    "trigger_source": "external_catalog_system",
    "event_type": "product_catalog_update",
    "timestamp": datetime.now().isoformat(),
    "external_job_id": "CAT_UPDATE_2024_001",
    "products_affected": ["P001", "P002", "P015", "P023"]
}

print(f"🌐 Webhook triggered by {webhook_context['trigger_source']}")
print(f"📅 Event: {webhook_context['event_type']}")

# COMMAND ----------
# Process webhook-specific data
affected_products = webhook_context["products_affected"]

if affected_products:
    # Process only affected products
    df_affected = spark.table("databricks_clar.bronze.products_bronze") \
        .filter(col("product_id").isin(affected_products))

    # Add webhook metadata
    df_with_webhook_info = df_affected.select(
        "*",
        lit(webhook_context["external_job_id"]).alias("external_job_id"),
        lit(webhook_context["event_type"]).alias("trigger_event"),
        current_timestamp().alias("webhook_processing_time")
    )

    # Update Silver layer for affected products only
    df_with_webhook_info.write \
        .format("delta") \
        .mode("overwrite") \
        .option("replaceWhere", f"product_id IN ({','.join([f\"'{p}'\"] for p in affected_products)})") \
        .saveAsTable("databricks_clar.silver.products_webhook_updates")

    processed_count = df_with_webhook_info.count()

else:
    # Full refresh triggered by webhook
    df_all = spark.table("databricks_clar.bronze.products_bronze")
    processed_count = df_all.count()

webhook_output = {
    "trigger_type": "webhook",
    "webhook_context": webhook_context,
    "products_processed": processed_count,
    "processing_mode": "incremental" if affected_products else "full_refresh"
}

dbutils.notebook.exit(json.dumps(webhook_output))

🚨 Exam Key Points

Cron Expression Patterns (Memorize These):

"0 0 2 * * ?"     # Daily at 2 AM
"0 0 * * * ?"     # Every hour
"0 */15 * * * ?"  # Every 15 minutes
"0 0 6 ? * MON-FRI"  # Weekdays at 6 AM
"0 0 1 1 * ?"     # Monthly (1st day, 1 AM)

Trigger Decision Matrix:

  • Predictable data arrival → Scheduled
  • Unpredictable file arrival → File arrival trigger
  • Real-time requirements → Continuous
  • External system events → Webhook
  • Development/testing → Manual

Common Exam Scenarios:

  • "Daily ETL at 3 AM Eastern" → Scheduled trigger with timezone
  • "Process files as they arrive" → File arrival trigger on volumes
  • "Real-time streaming pipeline" → Continuous trigger
  • "External system notification" → Webhook trigger
  • "Prevent overlapping runs" → max_concurrent_runs: 1

Gotchas:

  • ❌ Continuous jobs with timeout_seconds → Continuous jobs shouldn't have timeouts
  • ❌ File arrival without min_time_between_triggers → Can cause rapid re-triggering
  • ❌ Scheduled job without timezone → Uses cluster timezone (inconsistent)
  • ✅ Always specify timezone for scheduled triggers
  • ✅ Use queueing for resource contention: queue.enabled: true

The Rule: Match trigger type to data arrival pattern for optimal efficiency and cost.

This transforms static schedules into intelligent, event-driven pipelines that respond appropriately to your data's natural rhythm! 🫡
