The Cortana Overview

Dynamic values = data that flows between tasks at runtime. Think of it as your DAG's nervous system — each task can pass results, metrics, and decisions to downstream tasks. Critical for conditional logic, monitoring, and adaptive pipelines.

🧠 Core Concepts

Task Value Types

Value Type When Created Access Pattern Example Use Case
Task Values Task execution completes {{tasks.task_key.values.metric_name}} Row counts, quality metrics
Job Parameters Job start {{job.parameters.param_name}} Environment configs, dates
Task Parameters Task start Local to task Task-specific overrides
Run Information Job start {{job.run_id}}, {{job.start_time}} Audit trails, logging

# COMMAND ----------
# Your enhanced Bronze logic with dynamic outputs
from pyspark.sql.functions import *
import json

# Process your products data
df_products = spark.read.format("csv") \
  .option("header", True) \
  .option("inferSchema", True) \
  .load("/Volumes/databricks_clar/bronze/bronze_volume/products/")

# Calculate metrics
product_count = df_products.count()
null_products = df_products.filter(col("product_id").isNull()).count()
avg_price = df_products.agg(avg("price")).collect()[0][0]
categories = df_products.select("category").distinct().count()

# Price distribution analysis
price_stats = df_products.agg(
    min("price").alias("min_price"),
    max("price").alias("max_price"),
    avg("price").alias("avg_price"),
    count("*").alias("total_products")
).collect()[0]

# COMMAND ----------
# Process customers data  
df_customers = spark.read.format("csv") \
  .option("header", True) \
  .option("inferSchema", True) \
  .load("/Volumes/databricks_clar/bronze/bronze_volume/customers/")

customer_count = df_customers.count()
null_customers = df_customers.filter(col("customer_id").isNull()).count()
unique_domains = df_customers.select(regexp_extract(col("email"), "@(.+)", 1).alias("domain")) \
  .distinct().count()

# COMMAND ----------
# Write to Delta tables (your original logic)
df_products.write.format("delta").mode("append").saveAsTable("databricks_clar.bronze.products_bronze")
df_customers.write.format("delta").mode("append").saveAsTable("databricks_clar.bronze.customers_bronze")

# COMMAND ----------
# 🎯 KEY: Return dynamic values for downstream tasks
task_output = {
    # Data quality metrics
    "product_count": int(product_count),
    "customer_count": int(customer_count),
    "total_null_count": int(null_products + null_customers),
    "null_ratio": float((null_products + null_customers) / (product_count + customer_count)),

    # Business metrics
    "avg_product_price": float(avg_price),
    "min_price": float(price_stats["min_price"]),
    "max_price": float(price_stats["max_price"]),
    "category_count": int(categories),
    "unique_domains": int(unique_domains),

    # Processing metadata
    "bronze_status": "SUCCESS",
    "processing_timestamp": str(spark.sql("SELECT current_timestamp()").collect()[0][0]),

    # Conditional flags for downstream routing
    "quality_passed": (null_products + null_customers) < 10,
    "high_value_products": avg_price > 200,
    "sufficient_data": product_count >= 90 and customer_count >= 1000
}

# Exit with JSON - this becomes available to all downstream tasks
dbutils.notebook.exit(json.dumps(task_output))

# notebooks/02_quality_condition.py
import json

# 🎯 Access upstream task values
bronze_values = json.loads(dbutils.notebook.run("01_bronze_ingestion_with_values", 0))

product_count = bronze_values["product_count"]
null_ratio = bronze_values["null_ratio"]
sufficient_data = bronze_values["sufficient_data"]

# Complex business logic
quality_score = 100 - (null_ratio * 100)  # 100 = perfect, 0 = all nulls
meets_sla = product_count >= 90 and quality_score >= 95

# Multi-condition logic
if meets_sla and sufficient_data:
    routing_decision = "PROCEED_TO_SILVER"
elif product_count >= 50:
    routing_decision = "PROCEED_WITH_WARNING"  
else:
    routing_decision = "HALT_PIPELINE"

# Return values for downstream conditional routing
result = {
    "quality_score": quality_score,
    "meets_sla": meets_sla,
    "routing_decision": routing_decision,
    "upstream_metrics": bronze_values  # Pass through all upstream values
}

dbutils.notebook.exit(json.dumps(result))

🎯 Task 3: Silver Processing (Adaptive Logic)

# notebooks/03_silver_adaptive.py

# COMMAND ----------
import json

# 🔥 EXAM PATTERN: Access multiple upstream task values
bronze_metrics = "{{tasks.bronze_ingestion.values}}"  # Job parameter injection
quality_decision = "{{tasks.data_quality_gate.values.routing_decision}}"

# Parse the JSON values
bronze_data = json.loads(bronze_metrics) if isinstance(bronze_metrics, str) else bronze_metrics

product_count = bronze_data["product_count"]
avg_price = bronze_data["avg_product_price"]
high_value_flag = bronze_data["high_value_products"]

# COMMAND ----------
# Adaptive processing based on upstream values
if quality_decision == "PROCEED_TO_SILVER":
    # Full Silver processing
    df_products = spark.table("databricks_clar.bronze.products_bronze")

    # Dynamic enrichment based on upstream analysis
    if high_value_flag:
        # Add premium product analysis for high-value datasets
        df_silver = df_products.select(
            "*",
            when(col("price") > avg_price * 1.5, "Premium")
            .when(col("price") > avg_price, "Standard")
            .otherwise("Economy").alias("price_segment"),

            # Add margin analysis for high-value products
            (col("price") * 0.3).alias("estimated_margin"),
            lit("high_value_processing").alias("processing_mode")
        )
    else:
        # Standard processing for regular datasets
        df_silver = df_products.select(
            "*",
            when(col("price") > 200, "High")
            .when(col("price") > 100, "Medium")
            .otherwise("Low").alias("price_tier"),
            lit("standard_processing").alias("processing_mode")
        )

elif quality_decision == "PROCEED_WITH_WARNING":
    # Minimal processing with data quality flags
    df_products = spark.table("databricks_clar.bronze.products_bronze")
    df_silver = df_products.select(
        "*",
        lit("quality_warning").alias("data_quality_flag"),
        lit("minimal_processing").alias("processing_mode")
    )

else:
    # Emergency mode - create empty table with schema only
    schema = spark.table("databricks_clar.bronze.products_bronze").schema
    df_silver = spark.createDataFrame([], schema) \
        .select("*", lit("emergency_mode").alias("processing_mode"))

# COMMAND ----------
# Write with dynamic table name based on processing mode
processing_mode = df_silver.select("processing_mode").first()[0]
table_suffix = f"_{processing_mode}" if processing_mode != "standard_processing" else ""

silver_table = f"databricks_clar.silver.products_silver{table_suffix}"
df_silver.write.format("delta").mode("overwrite").saveAsTable(silver_table)

# COMMAND ----------
# Return Silver metrics for Gold layer
silver_metrics = {
    "silver_table": silver_table,
    "silver_count": df_silver.count(),
    "processing_mode": processing_mode,
    "upstream_avg_price": avg_price,
    "quality_decision": quality_decision
}

dbutils.notebook.exit(json.dumps(silver_metrics))

📈 Task 4: Gold Analytics (Dynamic Aggregation)

-- notebooks/04_gold_dynamic.sql
-- SQL task with dynamic value injection

-- 🎯 EXAM PATTERN: SQL tasks can access dynamic values via parameters

-- Create dynamic aggregation based on upstream processing mode
CREATE OR REPLACE TABLE databricks_clar.gold.product_analytics_${silver_processing_mode} AS
SELECT 
  category,

  -- Dynamic grouping based on upstream analysis
  CASE 
    WHEN '{{tasks.silver_processing.values.processing_mode}}' = 'high_value_processing' 
    THEN price_segment
    ELSE price_tier
  END as price_category,

  COUNT(*) as product_count,
  ROUND(AVG(price), 2) as avg_price,
  ROUND(MIN(price), 2) as min_price, 
  ROUND(MAX(price), 2) as max_price,

  -- Conditional metrics based on processing mode
  CASE 
    WHEN '{{tasks.silver_processing.values.processing_mode}}' = 'high_value_processing'
    THEN ROUND(AVG(estimated_margin), 2)
    ELSE NULL
  END as avg_margin,

  -- Metadata from entire pipeline
  '{{tasks.bronze_ingestion.values.processing_timestamp}}' as source_timestamp,
  '{{job.run_id}}' as job_run_id,
  current_timestamp() as gold_created_at

FROM ${silver_table}
WHERE processing_mode = '{{tasks.silver_processing.values.processing_mode}}'
GROUP BY 
  category,
  CASE 
    WHEN '{{tasks.silver_processing.values.processing_mode}}' = 'high_value_processing' 
    THEN price_segment
    ELSE price_tier
  END
ORDER BY category, price_category;

🚨 Task 5: Alert Handler (Comprehensive Values Access)

# notebooks/05_alert_comprehensive.py

# COMMAND ----------
import json
from datetime import datetime

# 🔥 EXAM FOCUS: Access ALL upstream task values for comprehensive alerting

# Method 1: Job parameters (injected by Job orchestration)
bronze_metrics_raw = dbutils.widgets.get("bronze_metrics")
quality_metrics_raw = dbutils.widgets.get("quality_metrics")
silver_metrics_raw = dbutils.widgets.get("silver_metrics")

# Parse JSON values
bronze_data = json.loads(bronze_metrics_raw)
quality_data = json.loads(quality_metrics_raw) 
silver_data = json.loads(silver_metrics_raw)

# COMMAND ----------
# Method 2: Direct task value access (in DAB configuration)
# These would be injected as: 
# "{{tasks.bronze_ingestion.values.product_count}}"
# "{{tasks.quality_gate.values.quality_score}}"
# "{{tasks.silver_processing.values.processing_mode}}"

job_run_id = dbutils.widgets.get("job_run_id")  # {{job.run_id}}
pipeline_start_time = dbutils.widgets.get("pipeline_start")  # {{job.start_time}}

# COMMAND ----------
# Comprehensive alert with all pipeline context
alert_details = {
    "alert_timestamp": datetime.now().isoformat(),
    "job_run_id": job_run_id,
    "pipeline_duration_minutes": (datetime.now() - datetime.fromisoformat(pipeline_start_time.replace('Z', '+00:00'))).total_seconds() / 60,

    # Bronze layer metrics
    "bronze_status": {
        "product_count": bronze_data["product_count"],
        "customer_count": bronze_data["customer_count"],
        "null_ratio": bronze_data["null_ratio"],
        "avg_price": bronze_data["avg_product_price"],
        "categories": bronze_data["category_count"]
    },

    # Quality assessment
    "quality_status": {
        "score": quality_data["quality_score"],
        "meets_sla": quality_data["meets_sla"],
        "routing": quality_data["routing_decision"]
    },

    # Processing outcome
    "silver_status": {
        "processing_mode": silver_data["processing_mode"],
        "table_created": silver_data["silver_table"],
        "record_count": silver_data["silver_count"]
    },

    # Derived insights
    "pipeline_health": "DEGRADED" if quality_data["quality_score"] < 90 else "HEALTHY",
    "data_volume_trend": "HIGH" if bronze_data["product_count"] > 150 else "NORMAL"
}

# COMMAND ----------
# Dynamic alerting based on pipeline state
if alert_details["pipeline_health"] == "DEGRADED":
    alert_severity = "HIGH"
    alert_message = f"""
🚨 PIPELINE DEGRADED - Immediate Attention Required
Quality Score: {quality_data['quality_score']:.1f}% (Target: 95%)
Processing Mode: {silver_data['processing_mode']}
Root Cause: {quality_data['routing_decision']}
"""
elif silver_data["processing_mode"] != "standard_processing":
    alert_severity = "MEDIUM" 
    alert_message = f"""
⚠️ NON-STANDARD PROCESSING - Monitor Required
Mode: {silver_data['processing_mode']}
Data Quality: {quality_data['quality_score']:.1f}%
Records Processed: {silver_data['silver_count']}
"""
else:
    alert_severity = "INFO"
    alert_message = f"""
✅ Pipeline Completed Successfully
Quality: {quality_data['quality_score']:.1f}%
Processing: Standard Mode
Records: {bronze_data['product_count']} → {silver_data['silver_count']}
"""

# COMMAND ----------
# Log comprehensive metrics to monitoring table
spark.sql(f"""
INSERT INTO databricks_clar.monitoring.pipeline_metrics VALUES (
  '{job_run_id}',
  '{datetime.now().isoformat()}',

  {quality_data['quality_score']},
  '{silver_data['processing_mode']}'

""")

# Return comprehensive result
dbutils.notebook.exit(json.dumps({
    "alert_sent": True,
    "severity": alert_severity,
    "pipeline_health": alert_details["pipeline_health"],
    "total_records_processed": bronze_data["product_count"] + bronze_data["customer_count"]

🎯 Exam Patterns & Gotchas

Value Access Syntax (Memorize These)

# Task output values
"{{tasks.task_name.values.metric_name}}"

# Job-level values  
"{{job.run_id}}"
"{{job.start_time}}" 
"{{job.parameters.param_name}}"

# Conditional execution
run_if: "{{tasks.upstream_task.values.boolean_flag}}"

# Comparison operators in condition tasks
op: "EQUAL_TO" | "NOT_EQUAL" | "GREATER_THAN" | "LESS_THAN" | "GREATER_THAN_OR_EQUAL" | "LESS_THAN_OR_EQUAL"

Common Exam Traps

  • ❌ {{task.bronze_ingestion.values.count}} → Missing 's' in 'tasks'
  • ❌ "{{tasks.my_task.outputs.metric}}" → Should be 'values', not 'outputs'
  • ❌ Using dynamic values in upstream tasks → Values only exist AFTER task completion
  • ❌ Condition task outcome: "success" → Should be outcome: "true" or outcome: "false"
  • ❌ JSON parsing in SQL tasks → SQL gets string values, use parameters for injection

Success Patterns

  • ✅ Always return JSON from notebook tasks for structured value passing
  • ✅ Use run_if: "ALL_DONE" for monitoring tasks that should always execute
  • ✅ Inject values as base_parameters for downstream tasks to access via dbutils.widgets.get()
  • ✅ Use condition tasks for simple boolean routing, notebook tasks for complex logic
  • ✅ Pass entire upstream value objects with "{{tasks.upstream.values}}" for comprehensive access
