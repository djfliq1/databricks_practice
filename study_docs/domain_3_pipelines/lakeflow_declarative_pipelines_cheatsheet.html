Lakeflow Declarative Pipelines

formerly Delta Live Tables · databricks_clar
EXAM DOMAIN: Data Processing 31%
Cortana TL;DR
  • Import: from pyspark import pipelines as dp — NOT import dlt anymore
  • Four table types: @dp.table (streaming), @dp.materialized_view (batch), staging view (temp), append flow (multi-source)
  • Dry Run = dev mode, no data published. Production = tables published to Unity Catalog
  • Expectations = data quality decorators — warn, drop, or fail per violation
  • Auto CDC = create_auto_cdc_flow() — replaces old apply_changes()
Import Pattern & Table Types
@dp.table — Streaming Table
from pyspark import pipelines as dp
from pyspark.sql.functions import col, upper, lit

@dp.table(name="bronze_products")
def bronze_products():
    return spark.readStream \
        .format("cloudFiles") \
        .option("cloudFiles.format", "csv") \
        .option("header", "true") \
        .option("inferSchema", "true") \
        .load("/Volumes/databricks_clar/bronze/autovol/raw/products/")
@dp.materialized_view — Batch
@dp.materialized_view(name="gold_category_summary")
def gold_category_summary():
    return spark.sql("""
        SELECT
          category,
          COUNT(*) as product_count,
          AVG(price) as avg_price,
          MAX(price) as max_price
        FROM databricks_clar.silver.products_enriched
        GROUP BY category
    """)
  • Batch / full refresh — no streaming
  • Cannot use @dp.table decorator for this
Staging View — Temp Transform
@dp.view(name="products_staging")
def products_staging():
    return spark.readStream \
        .option("skipChangeCommits", True) \
        .table("bronze_products") \
        .withColumn("category", upper(col("category"))) \
        .withColumn("flag", lit("processed"))

# Never published — temp pipeline step only
Append Flow — Multi-Source
@dp.table(name="silver_products")
def silver_products(): pass

@dp.append_flow(target="silver_products")
def load_from_bronze():
    return spark.readStream.table("bronze_products")

# Backfill (runs once)
@dp.append_flow(target="silver_products", once=True)
def backfill_legacy():
    return spark.read.table("old_products_archive")
Data Quality Expectations
Three Modes
DecoratorOn ViolationUse When
@dp.expectWARN — keep row, log metricMonitoring only, non-critical quality
@dp.expect_or_dropDROP — remove bad rowBad rows shouldn't pollute downstream
@dp.expect_or_failFAIL — stop the pipelineZero-tolerance, must-be-clean data
# Stack expectations on products table
@dp.table(name="silver_products_clean")
@dp.expect("price_positive", "price > 0")
@dp.expect_or_drop("product_id_not_null", "product_id IS NOT NULL")
@dp.expect_or_fail("no_negative_price", "price >= 0")
def silver_products_clean():
    return spark.readStream \
        .option("skipChangeCommits", True) \
        .table("bronze_products")

dp.create_auto_cdc_flow(
  source="bronze_products_cdc",
  target="silver_products_history",
  keys=["product_id"],
  sequence_by=col("processDate"),

Exam Traps
⚠ Trap 01
Import is from pyspark import pipelines as dp — NOT import dlt. The rebrand to Lakeflow changed the import path entirely.
⚠ Trap 02
@dp.table = streaming ONLY. If you need a batch/full-refresh table, use @dp.materialized_view. Mixing them up is a classic exam trap.
⚠ Trap 03
sequence_by requires col() — not a plain string. sequence_by="processDate" is WRONG. Use sequence_by=col("processDate").
⚠ Trap 04
once=True in append_flow = backfill. It runs exactly once as a batch load. After that, it's done — no future triggers.
⚠ Trap 05
Renaming a flow kills the checkpoint. If you rename an append_flow function, its checkpoint is orphaned and the flow starts fresh from the beginning.
✓ Key Point
Dry Run = sandbox only. No tables are published to Unity Catalog in Dry Run mode — it's purely for development and validation.
✓ Key Point
skipChangeCommits=True when reading a CDC-enabled Delta source into a streaming table. Without it, update/delete records cause errors.
