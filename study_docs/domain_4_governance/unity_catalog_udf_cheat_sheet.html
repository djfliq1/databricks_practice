SQL Scalar Functions

Core Syntax Pattern

CREATE OR REPLACE FUNCTION catalog.schema.function_name(param TYPE) RETURNS return_type RETURN sql_expression;

Common Patterns

CASE-based Categorization
CREATE FUNCTION databricks_clar.retail.price_tier(price DOUBLE) RETURNS STRING RETURN CASE WHEN price >= 400 THEN 'PREMIUM' WHEN price >= 200 THEN 'MID' ELSE 'BUDGET' END;
Mathematical Operations
CREATE FUNCTION apply_tax(price DOUBLE, tax_rate DOUBLE) RETURNS DOUBLE RETURN ROUND(price * (1 + tax_rate), 2);
String Manipulation
CREATE FUNCTION format_product_code(category STRING, product_id INT) RETURNS STRING RETURN CONCAT( UPPER(LEFT(category, 3)), '-', LPAD(CAST(product_id AS STRING), 6, '0') );

Performance Benefits

Catalyst Optimization

Query optimizer can inline and optimize SQL expressions

Pushdown Eligible

Can be pushed down to storage layer for better performance

No Serialization

No Python serialization overhead like Python UDFs

~20% Faster

Typically faster than equivalent Python UDFs

❌ SQL Scalar Limitations:
• Single expression only (no multi-statement blocks)
• No non-deterministic functions (RAND(), CURRENT_TIMESTAMP())
• No external calls or side effects
• Limited to SQL built-in functions

Python Scalar Functions

Runtime Requirements

DBR 13.3 LTS+

Minimum Databricks Runtime version required

Serverless/Shared

Must run on serverless or shared access mode clusters

Deterministic Only

No randomness, external calls, or side effects

Built-in Libraries

No package installations - built-in Python only

Core Syntax

CREATE OR REPLACE FUNCTION function_name(param TYPE) RETURNS return_type LANGUAGE PYTHON AS $$ # Python code here return result $$;

Available vs Unavailable Libraries

✅ Available
json datetime re math hashlib base64

String operations, timedelta calculations, basic math functions

❌ Not Available
requests pandas numpy external APIs file I/O pip installs

No external libraries or network access

Complex Example

CREATE FUNCTION databricks_clar.retail.calculate_loyalty_points( price DOUBLE, category STRING, is_member BOOLEAN ) RETURNS INT LANGUAGE PYTHON AS $$ base_points = int(price * 0.1) if category == "Electronics": multiplier = 3 if is_member else 2 elif category == "Books": multiplier = 2 if is_member else 1 else: multiplier = 1 return base_points * multiplier $$;
⚠️ Common Gotcha: Import statements go inside the function body, not at the top level. Runtime errors happen during execution, not function creation.

User Defined Table Functions

Scalar vs Table Functions

Scalar UDF

Pattern: input → single value
Usage: SELECT func(col) FROM table
Location: SELECT clause

Table UDF (UDTF)

Pattern: input → multiple rows
Usage: SELECT * FROM TABLE(func(val))
Location: FROM clause with TABLE() wrapper

Core Syntax

CREATE OR REPLACE FUNCTION explode_data(input STRING) RETURNS TABLE(col1 TYPE, col2 TYPE) LANGUAGE PYTHON AS $$ for item in input.split(','): yield (item.strip(), len(item)) $$;

Key Requirements

RETURNS TABLE Define output schema
yield keyword Not return
TABLE() wrapper Required in FROM clause
LATERAL VIEW For row expansion

Common Use Cases

String Explosion to Rows
-- Input: "Electronics,Books,Toys" -- Output: 3 rows with category and position CREATE FUNCTION explode_categories(category_list STRING) RETURNS TABLE(category STRING, position INT) LANGUAGE PYTHON AS $$ if not category_list: return categories = category_list.split(',') for i, category in enumerate(categories): yield (category.strip(), i + 1) $$;
Price Tier Generation
-- Input: 367.34 -- Output: All tiers with current flag SELECT p.product_name, p.price, t.tier_name, t.is_current FROM products p LATERAL VIEW TABLE(generate_price_tiers(p.price)) t;
❌ Common Mistakes:
• Using UDTF without TABLE() wrapper
• Using return instead of yield
• Calling UDTF in SELECT clause like scalar function
• Missing LATERAL VIEW for row expansion

Governance & Security

Permission Model

Catalog Level

USE CATALOG
Required to access anything in the catalog

Schema Level

USE SCHEMA
Required to see functions in the schema

Function Level

EXECUTE
Required to call the specific function

Common Commands

-- Grant function access GRANT EXECUTE ON FUNCTION catalog.schema.func_name TO `analysts`; -- Revoke access REVOKE EXECUTE ON FUNCTION catalog.schema.func_name FROM `user@company.com`; -- View permissions SHOW GRANTS ON FUNCTION catalog.schema.func_name; -- Inspect function DESCRIBE FUNCTION EXTENDED catalog.schema.func_name; -- Drop function DROP FUNCTION IF EXISTS catalog.schema.func_name;

Lifecycle Management

CREATE OR REPLACE Update existing safely
IF EXISTS Safe drop operations
COMMENT Function documentation
Audit Logs Usage tracking
📝 Exam Tip: Questions about "restricting access" or "only analysts can call" → GRANT EXECUTE. Questions about "function not found" → check USE CATALOG + USE SCHEMA permissions.

A) Any DBR version

A) SELECT udtf_name() FROM table

✅ B) SELECT * FROM TABLE(udtf_name()) - TABLE() wrapper required

A) USE CATALOG permission
B) USE SCHEMA permission
C) EXECUTE permission
D) MODIFY permission

✅ C) GRANT EXECUTE ON FUNCTION permission

Memory Tricks

"Session" vs "Unity Catalog"

Session = temporary, dies with cluster
Unity Catalog = permanent, metastore-backed

"SQL" vs "Python"

SQL = fast but limited (single expression)
Python = flexible but slower (multi-line)

"Scalar" vs "Table"

Scalar = one value output
Table = many rows output with yield

"DBR 13.3+"

Magic number for Python UDFs
+ serverless/shared clusters only
