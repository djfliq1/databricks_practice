Lakeflow Connect

Managed ingestion from SaaS & databases · databricks_clar
EXAM DOMAIN: Development & Ingestion 30%
Cortana TL;DR
  • Lakeflow Connect = managed ingestion. Point-and-click connectors from SaaS apps and databases into your Lakehouse
  • Three pillars: Local Files, Standard Connectors (Auto Loader), Managed Connectors (Fivetran-powered)
  • Lakeflow Connect ≠ Federation. Connect = data COPY into Lakehouse. Federation = zero-copy window into external data
  • Fivetran partnership = Managed Connectors. 300+ sources, fully managed, no infrastructure
  • 100 free DBUs/day per workspace (2026 update)
Three Pillars
Pillar 1: Local Files
  • Upload files directly via UI or API
  • CSV, JSON, Parquet, Avro, Excel
  • Landing in Volumes or DBFS
  • Best for: one-time loads, small datasets
  • No ongoing pipeline — manual operation
Pillar 2: Standard Connectors
  • Auto Loader (cloudFiles) — the core pattern
  • Incremental file ingestion from cloud storage
  • Structured Streaming under the hood
  • Schema inference + evolution
  • Best for: custom file-based pipelines
Pillar 3: Managed Connectors
  • Fivetran-powered, 300+ SaaS sources
  • Salesforce, HubSpot, Stripe, GitHub, etc.
  • Zero infrastructure — fully managed
  • Auto schema evolution + backfill
  • Best for: SaaS data, no ETL code needed

Managed Connector Workflow
Setup via UI — No Code Required
  • Step 1: Go to Data Ingestion → Lakeflow Connect in workspace UI
  • Step 2: Select connector (e.g., Salesforce, GitHub, Stripe)
  • Step 3: Configure source credentials and connection settings
  • Step 4: Select destination catalog and schema (e.g., databricks_clar.bronze)
  • Step 5: Choose sync mode: Full Refresh, Incremental Append, or Incremental Deduped History
  • Step 6: Set sync schedule and click Deploy — Databricks handles everything else
Managed Connectors auto-handle: schema drift detection · backfill · retry on failure · CDC · incremental state tracking
Sync Modes
Full Refresh
  • Truncate + reload entire table every sync
  • Simplest, always consistent
  • High cost at scale — moves all data every time
  • Use for: small, infrequently changing tables
Incremental Append
  • Only new rows synced each run
  • Source must have an ordering/timestamp column
  • No deletes or updates handled
  • Use for: append-only sources (logs, events)
Incremental Deduped
  • Incremental + handles updates via upsert
  • Uses primary key for deduplication
  • Best of both worlds for transactional sources
  • Use for: CRM, ERP, most SaaS apps
Exam Traps
⚠ Trap 01
Lakeflow Connect ≠ Federation. Connect brings data IN (copies it). Federation queries data IN-PLACE (zero copy). These are fundamentally different tools for different needs.
⚠ Trap 02
Managed Connectors = Fivetran under the hood. Databricks partners with Fivetran for managed connector infrastructure. The exam may reference either name.
⚠ Trap 03
Auto Loader = Standard Connector, not Managed. Auto Loader (cloudFiles) is the Standard Connector pattern for file-based ingestion. Managed Connectors are the SaaS/database GUI-based ones.
✓ Key Point
Decision framework: SaaS source (Salesforce, Stripe) → Managed Connector. File-based (CSV, JSON landing in cloud storage) → Auto Loader (Standard Connector). Query without copying → Federation.
✓ Key Point
100 free DBUs/day is the 2026 free tier for Lakeflow Connect. Roughly 100 million records/day at no charge. Exam may test awareness of this limit.
