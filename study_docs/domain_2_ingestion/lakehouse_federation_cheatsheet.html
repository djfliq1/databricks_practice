Lakehouse Federation

Zero-copy external data access · databricks_clar
EXAM DOMAIN: Governance 11%
Cortana TL;DR — The One Mental Model
  • Federation = window. You see external data in place — zero data copy, zero storage cost
  • Ingestion = copy. Auto Loader, COPY INTO — you move data into your Lakehouse
  • Read-only. You CANNOT INSERT, UPDATE, DELETE, or run DDL against federated sources
  • Three objects: Connection → Foreign Catalog → Query with USE CATALOG
  • Supported: PostgreSQL, MySQL, SQL Server, Snowflake, BigQuery, Redshift, SQLite, and more
Setup Workflow — Three Steps
Connection → Foreign Catalog → Query
-- STEP 1: Create Connection (store credentials)
CREATE CONNECTION postgres_prod
TYPE POSTGRESQL
OPTIONS (
  host 'prod-db.company.com',
  port '5432',
  database 'inventory',
  user secret('prod-scope', 'pg-user'),
  password secret('prod-scope', 'pg-pass')
);

-- STEP 2: Register as Foreign Catalog in Unity Catalog
CREATE FOREIGN CATALOG postgres_inventory
USING CONNECTION postgres_prod
OPTIONS (database 'inventory');

-- STEP 3: Grant access via UC RBAC
GRANT USE CATALOG, USE SCHEMA, SELECT
ON CATALOG postgres_inventory
TO `data-team`;

-- STEP 4: Query like any local table
SELECT p.product_id, p.product_name, i.stock_qty
FROM databricks_clar.silver.products_enriched p
JOIN postgres_inventory.public.inventory i
  ON p.product_id = i.product_id
WHERE i.stock_qty < 10;
What You Can and Cannot Do
Allowed Operations
  • SELECT — read federated tables
  • JOIN — join federated with local Delta tables
  • Aggregations — GROUP BY, COUNT, AVG etc.
  • Window functions — RANK, ROW_NUMBER etc.
  • Subqueries — federated tables as subquery source
  • CTAS — CREATE TABLE AS SELECT from federated → local Delta
NOT Allowed
  • INSERT — cannot write back to source
  • UPDATE — read-only, no modifications
  • DELETE — cannot remove source data
  • MERGE INTO — federated as target not supported
  • DDL — no CREATE TABLE, ALTER TABLE on federated
  • OPTIMIZE / VACUUM — only for Delta tables

Exam Traps
⚠ Trap 01
Federation is READ-ONLY. Any scenario asking you to write, update, or merge data back to a federated source — the answer is always: not possible. You need to ingest first.
⚠ Trap 02
Connection vs Foreign Catalog. Connection = credentials/endpoint. Foreign Catalog = the UC object that maps to the external database. You need both — one doesn't substitute for the other.
⚠ Trap 03
UC RBAC governs federated access. Even though data lives outside Databricks, access to federated catalogs is controlled by Unity Catalog GRANT/REVOKE — not by external database permissions alone.
✓ Key Point — The Exam Mental Model
Federation = window into external data (zero-copy). Ingestion = bringing data home (creating a copy). Every exam scenario involving "no data movement" = Federation. Every scenario involving "persist" or "transform and store" = Ingestion.
✓ Key Point
CTAS is the bridge. CREATE TABLE AS SELECT from a federated source to a local Delta table is how you convert a federated query into a persisted local copy when you need write access or performance optimization.
