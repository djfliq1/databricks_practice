🏗️ Basic AutoLoader Pattern

# AutoLoader streaming pattern df = spark.readStream.format("cloudFiles")\ .option("cloudFiles.format", "csv")\ .option("cloudFiles.schemaLocation", "/schemas/")\ .option("cloudFiles.schemaEvolutionMode", "addNewColumns")\ .load("/source/path/") # Write stream with checkpoint df.writeStream\ .format("delta")\ .option("checkpointLocation", "/checkpoint/")\ .option("mergeSchema", "true")\ .trigger(availableNow=True)\ .start("/destination/")

from pyspark.sql.functions import current_timestamp, input_file_name # Production-grade configuration bronze_df = spark.readStream\ .format("cloudFiles")\ .option("cloudFiles.format", "csv")\ .option("header", "true")\ \ # Schema management\ .option("cloudFiles.schemaLocation", bronze_schema_path)\ .option("cloudFiles.schemaEvolutionMode", "addNewColumnsWithTypeWidening")\ \ # Type inference control\ .option("cloudFiles.inferColumnTypes", "true")\ .option("cloudFiles.schemaHints", "product_id long, price decimal(10,2)")\ \ # Data safety\ .option("rescuedDataColumn", "_rescued_data")\ .option("cloudFiles.useNotifications", "true")\ \ # Load with metadata\ .load("/path/to/source/")\ .withColumn("_ingestion_timestamp", current_timestamp())\ .withColumn("_source_file", input_file_name())

📋 Basic COPY INTO Syntax

-- Simple CSV load COPY INTO main.bronze.products FROM '/path/to/csv/files/' FILEFORMAT = CSV FORMAT_OPTIONS ('header' = 'true', 'inferSchema' = 'true');

🔍 Validation Before Loading

-- Test first, then load COPY INTO main.bronze.products FROM '/path/to/files/' FILEFORMAT = CSV VALIDATE 100 ROWS; -- If validation passes, run actual load COPY INTO main.bronze.products FROM '/path/to/files/' FILEFORMAT = CSV FORMAT_OPTIONS ('header' = 'true');

🧬 Schema Evolution with COPY INTO

-- Create schemaless table (DBR 11.3+) CREATE TABLE IF NOT EXISTS main.bronze.products_evolved; -- Auto-evolve schema during copy COPY INTO main.bronze.products_evolved FROM '/path/to/files/' FILEFORMAT = CSV FORMAT_OPTIONS ('header' = 'true', 'mergeSchema' = 'true') COPY_OPTIONS ('mergeSchema' = 'true');

🎯 Selective File Loading

-- Load specific files COPY INTO main.bronze.orders FROM '/migration/orders/' FILEFORMAT = CSV FILES = ('orders_2023.csv', 'orders_2024.csv') FORMAT_OPTIONS ('header' = 'true'); -- Pattern matching COPY INTO main.bronze.orders FROM '/migration/orders/' FILEFORMAT = CSV PATTERN = 'order_*.csv';

🔄 Transformations During Copy

-- Apply transformations during copy COPY INTO main.bronze.products (product_id, product_name, category, price, load_date) FROM ( SELECT product_id, UPPER(product_name) as product_name, category, price, current_timestamp() as load_date FROM '/path/to/products/' ) FILEFORMAT = CSV FORMAT_OPTIONS ('header' = 'true');

🏆 Why COPY INTO is the "Migration King"

  • One-time bulk loads: Perfect for historical data migration
  • Idempotent operations: Restart safely without duplicates
  • Validation mode: Test before committing
  • SQL simplicity: No Python/Scala required
  • Selective reprocessing: Cherry-pick problematic files

🎯 Databricks Exam Success Guide

📝 Most Tested Scenarios

Scenario 1: "CSV files suddenly have a new column, AutoLoader failed with UnknownFieldException"
Answer: Schema was updated before error. Just restart the stream.
Scenario 2: "Pipeline failed halfway through 1000 files. Will restart reprocess all files?"
Answer: No. AutoLoader's RocksDB tracks processed files. Only remaining files processed.
Scenario 3: "Getting millions of files/hour. Directory listing is slow."
Answer: Switch to file notification mode with cloud storage events.
Scenario 4: "Need to migrate 10TB historical data + daily increments."
Answer: COPY INTO for migration, AutoLoader for daily ops.

🔑 Key Concepts to Remember

🗂️ File Locations

  • checkpointLocation: Streaming state (offsets)
  • schemaLocation: Schema versions
  • source: Raw files
  • destination: Delta table

⚡ AutoLoader Sampling

  • Samples first 50GB or 1000 files
  • CSV/JSON inferred as strings
  • Parquet/Avro uses file metadata
  • Schema stored in _schemas directory

🔄 Discovery Modes

  • Directory Listing: Periodic scans
  • File Notification: SNS/SQS events
  • Notification = millions of files/hour
  • Directory = <10K files

📋 COPY INTO Features

  • Idempotent: No duplicates
  • Validation: Test before load
  • Selective: FILES/PATTERN matching
  • SQL-native: No Python required
