📊 Delta Lake Deep Dive

Delta Lake = Open table format on top of Parquet files

The transaction log is the file that stores all transactions

Transaction Log Structure

_delta_log/ ├── 00000000000000000000.json ← Transaction 0 (table creation) ├── 00000000000000000001.json ← Transaction 1 (first INSERT) ├── 00000000000000000002.json ← Transaction 2 (UPDATE) ├── 00000000000000000003.json ← Transaction 3 (DELETE)

Transaction File Contents

{ "commitInfo": { "timestamp": 1640995200000, "operation": "WRITE", "operationParameters": {"mode": "Append"} }, "add": { "path": "part-00001.parquet", "size": 1024, "modificationTime": 1640995200000, "dataChange": true } }

ACID Operations

-- Delete operations DELETE FROM orders WHERE order_date < '2023-01-01'; -- Updates UPDATE orders SET status = 'shipped' WHERE id = 123; -- Upserts (Merge) MERGE INTO customers USING new_data ON customer_id = id WHEN MATCHED THEN UPDATE SET * WHEN NOT MATCHED THEN INSERT *; -- Time Travel SELECT * FROM orders VERSION AS OF '2024-01-01'; SELECT * FROM orders TIMESTAMP AS OF '2024-01-01 10:00:00';

In-Memory Caching Example

# Cache hot datasets in memory df = spark.read.parquet("s3://huge-dataset/") df.cache() # or df.persist(StorageLevel.MEMORY_AND_DISK) # All subsequent operations use cached data result1 = df.filter("status = 'active'").count() # Cache miss - loads to memory result2 = df.groupBy("region").sum("revenue") # Cache hit - blazing fast result3 = df.join(other_df, "key") # Cache hit - instant

Bronze Layer - Data Dump

Purpose: Ingest everything, break nothing

# Just land it - deal with schema later bronze_events = (spark.readStream .format("cloudFiles") .option("cloudFiles.format", "json") .load("/raw/events/") .writeStream.format("delta") .option("checkpointLocation", "/checkpoints/bronze/") .start("/lakehouse/bronze/events/") )

Silver Layer - Data Cleaning Factory

Purpose: Clean, standardize, dedupe

# Clean and conform silver_events = (bronze_events .filter("event_type IS NOT NULL") .withColumn("timestamp", to_timestamp("event_time")) .dropDuplicates(["user_id", "event_id"]) .withColumn("user_id", regexp_replace("user_id", "[^0-9]", "")) )

Gold Layer - Business Logic & Modeling

Purpose: Business metrics, aggregations, star/snowflake schemas

# Business-ready aggregations gold_daily_metrics = (silver_events .groupBy( date_trunc("day", "timestamp").alias("date"), "product_category" ) .agg( count("*").alias("total_events"), countDistinct("user_id").alias("unique_users"), sum("revenue").alias("daily_revenue") ) )

🔄 Delta Live Tables (DLT) / Declarative Pipelines

Declarative ETL: Define WHAT you want, not HOW to build it

May have been renamed to "Declarative Pipelines"

DLT Approach

import dlt @dlt.table def bronze_orders(): return spark.readStream.format("cloudFiles").load("/raw/orders/") @dlt.table def silver_orders(): return dlt.read("bronze_orders").filter("order_value > 0") @dlt.table def gold_daily_revenue(): return (dlt.read("silver_orders") .groupBy(date_trunc("day", "order_date")) .sum("order_value"))

DLT Benefits

🛠️ Common Patterns & Best Practices

Schema Evolution

# Delta Lake automatic schema evolution df_new.write.mode("append").option("mergeSchema", "true").delta("/delta/table") # Handle schema changes gracefully df = spark.read.format("delta").load("/delta/table") if "new_column" not in df.columns: df = df.withColumn("new_column", lit(None))

Delta Lake Maintenance

from delta.tables import DeltaTable # Optimize (compaction) DeltaTable.forPath(spark, "/delta/table").optimize().executeCompaction() # Vacuum (cleanup old files) DeltaTable.forPath(spark, "/delta/table").vacuum(168) # 7 days retention # Z-Order optimization DeltaTable.forPath(spark, "/delta/table").optimize().executeZOrderBy("customer_id")

Data Quality Patterns

from pyspark.sql.functions import * # Find duplicates duplicates = df.groupBy("customer_id").count().filter("count > 1") # Null analysis null_analysis = df.select([ count(when(col(c).isNull(), c)).alias(c + "_nulls") for c in df.columns ]) # Data quality expectations in DLT @dlt.expect("valid_email", "email RLIKE '^[^@]+@[^@]+\\.[^@]+$'") @dlt.expect_or_drop("positive_amount", "amount > 0")
